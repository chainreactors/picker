著名开源社区平台工具 Discourse 3.0.0 beta 发布

**Discourse** 是一款开源社区工具，「为互联网的下一个十年而构建的 100% 开源讨论平台。将其用作邮件列表、论坛、长篇聊天室等」，目前已发布 Discourse 3.0.0.beta15 版本，新增 侧边栏、用户菜单、实时聊天 等内置功能。@Appinn

![著名开源社区平台工具 Discourse 3.0.0 beta 发布](https://static1.appinn.com/images/202301/discourse-3-0.jpg!o "著名开源社区平台工具 Discourse 3.0.0 beta 发布 1")

目前的[小众软件论坛](https://meta.appinn.net)（<https://meta.appinn.net>）就在使用 Discourse，各位感兴趣的话，可以去看看，样式基本默认，和官网论坛基本上一直。

## Discourse 3.0.0 beta15

Discourse 3.0.0 beta15 从 2.9.0.beta14 升级而来，主要包括两个升级：

* 🆕 [侧边栏 + 新用户菜单](https://meta.discourse.org/t/try-out-the-new-sidebar-and-notification-menus/238821)
* 🆕 [实时聊天](https://meta.discourse.org/t/discourse-chat/230881)

实际上这两个功能早在 2.9.0 测试中就已经内置，不过需要论坛管理员手动打开。如今的 Discourse 3.0.0 则是在安装过程中，由管理员选择是否打开：

![著名开源社区平台工具 Discourse 3.0.0 beta 发布 1](https://static1.appinn.com/images/202301/8c1e76d3dd6ecc41f9091c49d7e1b2fbf474786e.jpg!o "著名开源社区平台工具 Discourse 3.0.0 beta 发布 2")

除了主要功能，还有不少新功能：（以下新功能可以忽略不看）：

* 警告管理员私人群组名称暴露给匿名用户
* 添加用户切换，以掩盖/取消掩盖密码
* 嵌入 Motoko 的 Onebox
* 在核心部分增加对桌面推送通知的支持
* 对标签进行排序，从优先级较高的术语开始
* 区分直接与间接观看的主题的通知类型
* 允许分类管理人设置自动删除主题的计时器
* 在惩罚一个用户时显示类似的用户
* 可选择允许单独指定 s3\_asset\_cdn\_url
* 允许显示没有术语的标签自动完成结果
* 更好的用户界面来管理2fa
* 强制执行聊天信息的提及限制
* 在帖子的内联提及上显示实时用户状态
* 支持Youtube Short onebox
* 允许UserApiKey的user\_status范围
* 改进作曲家对提及的警告
* 频道可以允许/不允许@all/@here提及
* 添加实验性插件API来注册消息导航下拉菜单
* 为新的PM和可审查的内容提供单独的通知指标

以上新功能可以忽略不看，因为 Discourse 的功能实在是太多了，现如今青小蛙在逛[小众软件论坛](https://meta.appinn.net/)的时候，还经常发现一些新功能，通常会收录到 #[论坛使用指南](https://meta.appinn.net/tag/%E8%AE%BA%E5%9D%9B%E4%BD%BF%E7%94%A8%E6%8C%87%E5%8D%97) 之中。
