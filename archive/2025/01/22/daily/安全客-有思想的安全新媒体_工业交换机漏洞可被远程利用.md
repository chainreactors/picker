# 工业交换机漏洞可被远程利用

![Industrial Switch Vulnerabilities Allow Remote Exploitation]()

攻击者可以利用工业网络交换机中的关键漏洞获得远程控制，从而入侵自动化系统、物联网设备和监控网络。

Claroty’s Team82 在 Planet Technology 制造的 WGS-804HPT 交换机中发现了三个漏洞，这些交换机通常部署在楼宇和家庭自动化网络中。这些设备为物联网系统、IP 监控摄像机和无线局域网应用提供了连接便利，并配备了基于 Web 的管理界面和 SNMP 支持。利用这些漏洞，攻击者可以执行任意命令、中断操作或在未经授权的情况下访问敏感网络。

Claroty 向 Planet Technology 报告了这些问题，Planet Technology 发布了修补过的固件更新。

Team82 复制了交换机的 MIPS 32 位架构，在受控环境下剖析了其固件，并检查了嵌入式网络服务器，该服务器通常是攻击者的切入点。

研究人员在交换机网络管理界面的 dispatcher.cgi 组件中发现了关键漏洞。这些漏洞可通过恶意 HTTP 请求加以利用，允许攻击者在未经身份验证的情况下入侵设备。

其中一个问题被追踪为 CVE-2024-48871，是一个基于堆栈的缓冲区溢出，CVSS 得分为 9.8。它可以远程执行代码，让攻击者在未经授权的情况下控制设备。第二个漏洞被跟踪为 CVE-2024-52320，CVSS 得分为 9.8，涉及操作系统命令注入漏洞，允许未经认证的攻击者执行任意命令。第三个漏洞被追踪为 CVE-2024-52558，CVSS 得分为 5.3，它是一个整数下溢漏洞，可利用畸形 HTTP 请求使设备崩溃，导致服务中断。这些发现凸显了关键自动化网络中未打补丁设备的重大风险。

CGI 脚本是处理发送到交换机网络接口的 HTTP 请求的中央调度程序，是设备管理功能的重要组成部分。

研究人员发现，该脚本缺乏适当的输入验证机制，因此特制的有效载荷可以绕过身份验证检查。

dispatcher.cgi 中最严重的漏洞之一源于对 URL 参数的不当处理。通过向特定参数注入恶意输入，攻击者可以利用命令注入漏洞，以 root 权限执行任意系统命令。

由于对用户提供的数据处理不当，交换机还容易受到反射式跨站脚本攻击。攻击者可利用该漏洞向 Web 界面注入恶意 JavaScript 代码，从而可能危及已通过身份验证的管理员的会话。
