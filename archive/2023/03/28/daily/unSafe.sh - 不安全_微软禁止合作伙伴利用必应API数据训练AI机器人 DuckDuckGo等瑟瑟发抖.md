微软旗下的必应搜索虽然市场份额远不如谷歌，但与谷歌等搜索引擎相比，必应开放 API 接口允许合作伙伴调用必应 API 获取搜索数据。

以 DuckDuckGo 为例：该搜索引擎没有自己的爬虫 (Spider / Bot)，不会自己从整个互联网里抓爬数据，而是通过必应 API 获取数据，必应自己有爬虫无时不刻不在互联网上抓爬数据。

使用必应 API 的好处是其他搜索引擎不需要自己开发爬虫，也不需要建立复杂的内容安全机制，只要向微软支付费用就可以获得关键词搜索数据，然后搭配自己的产品添加关键词广告。

但把搜索引擎的核心之一交给别人做，那就有可能在某个时候引起问题，比如现在微软禁止合作伙伴利用必应 API 抓取数据训练自己的 AI 机器人。

[![微软禁止其他AI机器人使用必应搜索数据 DuckDuckGo等瑟瑟发抖](https://img.lancdn.com/landian/2023/03/97896.png)](https://img.lancdn.com/landian/2023/03/97896.png)

**微软打击竞争对手：**

目前在 AI 机器人领域主要就是微软对阵谷歌，而且微软现在有巨大的优势，Bing Chat 首发优势已经成功让必应搜索市场份额提高 1 个百分点。

微软当然想要保持这种优势，虽然没法控制谷歌，但可以控制那些使用必应 API 的中小型搜索引擎提供商。

彭博社报道称微软至少已经向两家合作伙伴发出通知，禁止这些合作伙伴利用必应 API 获取的数据训练、开发自己的 AI 机器人，这些提供商可以自己开发 AI 机器人但前提是自己开发爬虫抓取数据，不能利用必应 API 的数据。

如果继续使用必应 API 数据训练、开发 AI 机器人则微软会取消合同，禁止他们再使用必应 API。

报道虽然没有明确指出这两家搜索引擎提供商是谁，但其中有一个是 Neeva 搜索引擎，另一个是谁不知道，最近 DuckDuckGo 也发布了自己的 AI 服务，不知道另一个是不是 DuckDuckGo。

从商业角度来说微软这个做法倒是不新鲜，你不能利用我的数据开发我产品的竞品，这种限制以前在 Facebook 上非常常见，当时的 FB 经常利用这些条款限制合作商。

**微软也倒是就此消息发布了一份回应，不过没有实质内容，微软发言人表示：**

我们一直在与不合规的合作伙伴保持联系，因为我们会继续全面执行我们的条款... 我们将继续直接与他们合作，并提供寻找前进道路所需的任何信息。
