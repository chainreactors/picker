Great news for the SAP Cloud Identity Services administrators : you can [now](https://roadmaps.sap.com/board?PRODUCT=67837800100800007337&range=CURRENT-LAST#;INNO=6EAE8B28C5D91EDB819666C60E5980ED) configure and work with the Identity Provisioning in the SAP Cloud Identity Services administration console !

The prerequisites are :

* The Identity Provisioning tenant must run on the SAP Cloud Identity Services infrastructure. Should your tenant still run on the SAP BTP, Neo environment, you have to migrate to SCI first. If you are unsure how, you will find¬† just the right guidance in our colleague's blog : [Migrate Identity Provisioning Bundle Tenant](https://help.sap.com/docs/IDENTITY_PROVISIONING/f48e822d6d484fa5ade7dda78b64d9f5/055f92d5e11d491c97b68223c68179ad.html).

* The administrator must have the Manage Identity Provisioning authorizations assigned in the SAP Cloud Identity Services console.

![](/legacyfs/online/storage/blog_attachments/2023/03/IPS-in-the-admin-concole-1-1.jpg)

Harmonized admin Web interface

Accessing the Identity Provisioning functionality is possible either from the Home section or from the drop menu from the horizontal panel.

![](/legacyfs/online/storage/blog_attachments/2023/03/Screenshot-2023-03-31-at-22.18.30.png)

Identity Provisioning

With this change, the SAP Cloud Identity Services administrators can manage all the configurations in one place, without having to switch between consoles (<https://<ias-host>/admin> ‚Üí¬†https://<ias-host>/ips).
