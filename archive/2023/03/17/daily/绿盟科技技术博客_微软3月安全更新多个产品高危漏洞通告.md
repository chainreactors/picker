# 微软3月安全更新多个产品高危漏洞通告

## ****一、漏洞概述****

3月15日，绿盟科技CERT监测到微软发布3月安全更新补丁，修复了82个安全问题，涉及Windows Hyper-V、Microsoft Outlook、Windows HTTP Protocol Stack、Microsoft Graphics、Microsoft Excel等广泛使用的产品，其中包括权限提升、远程代码执行等高危漏洞类型。

本月微软月度更新修复的漏洞中，严重程度为关键（Critical）的漏洞有9个，重要（Important）漏洞有69个，其中包括2个0day漏洞：

Microsoft Outlook 特权提升漏洞（CVE-2023-23397）

Windows SmartScreen 安全功能绕过漏洞（CVE-2023-24880）

请相关用户尽快更新补丁进行防护，完整漏洞列表请参考附录。

参考链接：

https://msrc.microsoft.com/update-guide/releaseNote/2023-Mar

## ****二、重点漏洞简述****

根据产品流行度和漏洞重要性筛选出此次更新中包含影响较大的漏洞，请相关用户重点进行关注：

****Microsoft Outlook权限提升漏洞（CVE-2023-23397）：****

Microsoft Outlook存在权限提升漏洞，未经身份验证的攻击者通过发送特制的电子邮件，导致受害者连接到攻击者控制的外部的UNC位置，造成受害者的Net-NTLMv2散列泄露给攻击者，后续攻击者可以将其中继到另一个服务并作为受害者进行身份验证，最终实现权限提升。此外，微软官方提示：攻击者所发送的特制邮件在 Outlook客户端检索和处理时可自动触发。目前已监测到该漏洞存在在野利用，CVSS评分为9.8。

官方通告链接：

https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2023-23397

****Windows SmartScreen安全功能绕过漏洞（CVE-2023-24880）：****

Windows SmartScreen中存在安全功能绕过漏洞，未经身份验证的远程攻击者通过诱导用户打开恶意文件，成功利用该漏洞的攻击者可以逃避Web标记（MOTW）防御，从而导致MOTW标记的安全功能的完整性和可用性受损。目前该漏洞已公开披露，且已监测到存在在野利用，CVSS评分为5.4。

官方通告链接：

https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2023-24880

****Internet Control Message Protocol（ICMP）远程代码执行漏洞（CVE-2023-23415）：****

Internet Control Message Protocol（ICMP）中存在远程代码执行漏洞，当目标主机运行绑定到原始套接字的应用程序时，未经身份验证的远程攻击者可向目标主机发送低级协议错误来利用此漏洞，最终实现在目标系统上任意执行代码，CVSS评分为9.8。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23415

****HTTP协议栈远程执行代码漏洞（CVE-2023-23392）：****

HTTP协议栈（HTTP.sys）存在远程执行代码漏洞。当服务器启用HTTP/3，且使用缓冲I/O时，未经身份验证的攻击者通过向目标服务器发送特制的HTTP数据包，从而实现执行任意代码，CVSS评分9.8。

Windows HTTP 协议栈（HTTP.sys）是Windows操作系统中处理HTTP请求的内核驱动程序，常见于Web浏览器与 Web 服务器之间的通信，以及Internet Information Services (IIS)中。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23392

****Remote Procedure Call Runtime远程代码执行漏洞（CVE-2023-21708）：****

Remote Procedure Call Runtime中存在远程代码执行漏洞，未经身份验证的远程攻击者通过向RPC主机发送特制的RPC调用，最终可实现在服务器端以与RPC服务相同权限任意执行代码。此外，微软官方建议在企业外围防火墙上阻止TCP 135端口，从而来降低该漏洞的潜在攻击。CVSS评分为9.8。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-21708

****Windows Cryptographic Services远程代码执行漏洞（CVE-2023-23416）：****

Windows Cryptographic Services中存在远程代码执行漏洞，攻击者首先需将恶意证书上传至处理或导入证书的服务中，也可通过说服经过身份验证的用户在自身的系统上导入恶意证书。无需身份验证的攻击者利用该漏洞最终可实现在目标系统上以用户权限执行任意代码，CVSS评分为8.4。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23416

****Windows Point-to-Point Tunneling Protocol远程代码执行漏洞（CVE-2023-23404）：****

Windows点到点隧道协议存在远程代码执行漏洞，未经身份验证的远程攻击者通过向RAS服务器发送特制连接请求，最终可实现在服务器端远程代码执行，且无需用户交互。CVSS评分为8.1。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23404

****Windows Hyper-V拒绝服务漏洞（CVE-2023-23411）：****

Windows Hyper-V是Microsoft的本地虚拟机管理程序。该产品中存在拒绝服务漏洞，具有低权限的攻击者可通过利用该漏洞，导致Hyper-V主机遭受拒绝服务攻击，CVSS评分为6.5。

官方通告链接：

https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23411
