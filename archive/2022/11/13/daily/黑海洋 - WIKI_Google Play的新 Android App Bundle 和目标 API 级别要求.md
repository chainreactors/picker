# Google Play的新 Android App Bundle 和目标 API 级别要求

发布时间:
2022-11-12

分类:
[共享资源/Free](https://blog.upx8.com/Free/)

热度:
9038

2022 年，我们将继续进行年度目标 API 级别更新，要求新应用在 8 月和 11 月针对所有应用更新的目标 API 级别 30 (Android 11)。此外，正如今年早些时候宣布的那样，Google Play 将要求新应用使用Android App Bundle发布格式。这为更多用户和开发人员带来了更小应用程序和更简单版本的好处，并支持对高级分发的持续投资。

超过 750,000 款应用和游戏已经使用 app bundle 在 Google Play 上发布到正式版。与通用 APK 相比，热门应用切换平均节省 15% 的大小。用户受益于较小的下载量，而Netflix和Riafy等开发者的安装成功率更高，这在入门级设备较多且数据速度较慢的地区尤其具有影响力。开发者切换可以使用高级分发功能，例如Play Asset Delivery和Play Feature Delivery。我们重视您的反馈，并计划在转换之前为Play 应用签名和 Android 应用捆绑引入更多功能和选项。

### 新应用的要求

从**2021**年**8 月起**，Google Play 管理中心将要求所有**新应用**：

* 使用**Android App Bundle**格式发布。
* 利用**Play Asset Delivery**或**Play Feature Delivery**交付超过 150MB 下载大小的资产或功能。新应用程序将不再支持扩展文件 (OBB)。
* **目标 API 级别 30** (Android 11) 或更高版本并根据行为变化进行调整。

### 更新现有应用程序的要求

从**2021**年**11 月起**，现有应用的更新将需要针对 API 级别 30 或更高级别，并根据 Android 11 中的行为变化进行调整。未接收更新的现有应用不受影响，可以继续从 Play 商店下载。

### 即时体验的要求

切换到 Android App Bundle 交付还将影响使用旧版 Instant 应用 ZIP 格式的即时体验。从 2021 年 8 月起，发布支持免安装的应用程序包将需要新的免安装体验*和*对现有免安装体验的更新。

### 一起前行

以下是所有更改的摘要：

|  |  |  |
| --- | --- | --- |
| **发布类型** | **已更换** | **需要 2022年 8 月** |
| Google Play 上的新应用 | APK | Android 应用程序包 (AAB) |
| 目标 API 级别设置为 29+ | 目标 API 级别设置为 30+ |
| 扩展文件 (OBB) | Play 资产交付 或 Play 功能交付 |
| **发布类型** | **已更换** | **需要 2022年 11 月** |
| Google Play 上现有应用的更新 | *没有新的发布格式要求* |
| 目标 API 级别设置为 29+ | 目标 API 级别设置为 30+ |

Wear OS 应用不受新的目标 API 级别要求的约束。

应用程序仍然可以使用 any `minSdkVersion`，因此您为旧 Android 版本构建应用程序的能力没有变化。

要了解有关过渡到 app bundle 的更多信息，请观看我们的新视频系列：现代 Android 开发 (MAD) 技能。我们非常感谢所有已经采用 app bundle 和 API 级别 30 的开发人员。我们期待与您一起推进 Android 平台。
