**关键词**

广告拦截

早前知名广告拦截软件 AdGuard 官方拦截规则存储库就曾被 Github 封锁过，被封锁后所有安装 AdGuard 的用户无法通过 Github 更新最新的拦截规则。

这里需要强调的是触发此类封锁似乎是某种自动化规则导致的，而不是 Github 因为版权或其他问题人工审查了拦截规则存储库。

虽然此前的拦截规则存储库被误封问题很快就被解决，但还有个问题持续困扰着 AdGuard，那就是偶发性的再次拦截。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/aBHpjnrGylhyYqxp8icDLic11UKaSMVQvvibMQqynEHblcLe84gGhj50wzVzOQq72aRTiaAdpMcicbUvoIY4SfMuNVA/640?wx_fmt=png&from=appmsg)

据 AdGuard 发布的消息，Github 早前再次封锁了部分拦截规则的存储库，此次被封锁的存储库包括针对韩国地区的 List-KR 拦截规则。

封锁时间可以追溯到两周前，包括一名主要贡献者的账户被标记为 SPAM 而暂停，随后 AdGuard 提交了恢复申请但已经两周还未收到回复。

而针对韩国地区的广告过滤规则 List-KR 由于被封锁后至今没有解封，这导致贡献者、用户和维护者无法添加新的广告拦截规则，终端用户也无法接收最新规则。

AdGuard 也算是业界知名的软件开发商，碰到这种问题提交工单也没能获得及时处理，最终还是要靠在 X/Twitter 上发帖增加影响力。

不少网友一边 Fxxk Github 一边建议 AdGuard 迁移到 Gitlab 等平台，这可能算是解决问题的一种途径，相对来说 Gitlab 出现这种错误封禁的概率较低以及假如误封后，人工处理速度也比较快。

但考虑到用户、贡献者以及维护者可能都更习惯于 Github 平台，所以迁移工作也不是说做就做的，暂时还不清楚 AdGuard 是否有迁移的想法。
