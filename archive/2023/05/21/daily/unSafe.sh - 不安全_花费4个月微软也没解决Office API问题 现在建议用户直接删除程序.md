1 月份的时候微软接到反馈发现 Windows 10/11 搜索栏和 UWP 应用程序可能无法正常打开或使用，经过调查微软确认该问题与 Windows Shell 和 Office API 有关，这些问题导致注册表项或数据出现异常。

而通过 Office API 集成的第三方应用程序可能也无法正常使用，包括利用 Office API 集成到 Windows、Office、Outlook 和 Outlook 日历的第三方应用可能都无法使用。

[![花费4个月微软也没解决Office API问题 现在建议用户直接删除程序](https://img.lancdn.com/landian/2023/05/98713.png)](https://img.lancdn.com/landian/2023/05/98713.png)

这个问题与 Windows 更新没有关系，尽管微软已经花费几个月尝试进行修复，但现在依然无法解决问题。为此微软在[健康仪表盘](https://learn.microsoft.com/en-us/windows/release-health/status-windows-11-22H2#start-menu--windows-search-and-uwp-apps-might-have-issues-opening)中发布公告，让用户卸载这些受影响的第三方应用程序。

微软表示要缓解此问题，用户可以卸载相关的集成应用，如果用户需要使用这些程序，则可以考虑联系第三方软件开发商，看看是否有解决办法。

这种情况确实比较奇怪，尽管 Windows 和 Office 经常出问题，但理论上修复肯定是可以修复的，不知道微软为什么选择放弃继续修复而是让用户卸载，如果第三方软件开发者也无法解决问题，那最终结果就是用户没法使用了。
