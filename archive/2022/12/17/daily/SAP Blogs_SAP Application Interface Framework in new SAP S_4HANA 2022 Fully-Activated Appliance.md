With SAP S/4HANA 2022, we have added **mass error handling** capability to the Fiori message monitors of SAP Application Interface Framework. It's actually supported in the Message Monitor for Integration Experts, see [Help Portal](https://help.sap.com/docs/ABAP_PLATFORM_NEW/4db1676c3f114f119b500bd80ccd944d/a46dc4f2980747838bed1492c8b95d17.html). With this new capability, you can edit and fix multiple erroneous messages at once.

If you like to try this out and your system is not yet on the latest release 2022, you may setup your own pre-configured SAP S/4HANA 2022 sandbox based on the latest **SAP S/4HANA 2022 Fully-Activated Appliance** solution on SAP Cloud Appliance Library.

Here, we have set up a couple of demo scenarios that you can run through to get familiar with the message monitoring capabilities in SAP S/4HANA. In the latest appliance, we have enhanced the demo guide with a step by step description about how to use the mass error handling.

If you like to find out more about the SAP Application Interface Framework demo scenarios in the appliance, check out my blog [SAP Application Interface Framework in the SAP S/4HANA Fully-Activated Appliance](https://blogs.sap.com/2021/12/22/sap-application-interface-framework-in-the-sap-s-4hana-fully-activated-appliance/). This however is based on the 2021 release.

You can access the **latest 2022 appliance** from [here](https://cal.sap.com/catalog#/applianceTemplates/983008db-db92-4d4d-ac79-7e2afa95a2e0).

The **updated demo guide** can be accessed from [here](https://www.sap.com/documents/2022/12/d4ddfdf4-547e-0010-bca6-c68f7e60039b.html).
