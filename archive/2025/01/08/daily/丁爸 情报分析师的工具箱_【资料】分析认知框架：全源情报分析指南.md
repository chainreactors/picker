《分析认知框架（FAC）：全源情报分析指南》，由Shim Enterprise, Inc. 和美国空军研究实验室（AFRL）联合撰写。该指南旨在为情报分析师提供一个系统且灵活的指导框架，以应对全源情报分析中遇到的复杂、非结构化和高度迭代的工作。报告涵盖了从分析师个体认知到情报问题陈述开发、分析计划制定、信息计划制定、信息获取以及分析技术等多个方面的内容。

+ 框架分析认知（FAC）的设计目的和背景
+ FAC旨在提供一个系统化但灵活的指南，用于进行全源情报分析，以适应其非结构化和高度迭代的工作性质。
+ FAC专注于帮助分析师深入理解研究对象的能力和意图，这些能力和意图受到人类决策和行动的影响。
+ FAC利用了可能适用于情报分析的科学和专业领域的现有知识，并在开发和评估过程中继续系统性地审查文献。
+ FAC结合了基于文献和调查人员领域专业知识的新知识和工作辅助工具。

+ 分析师作为个体的角色和思考方式
+ 第3章探讨了分析师作为个体的角色，强调了思考方式的重要性。
+ 包括了关于思维过程、心智功能、思维倾向、心智软件、信念、知识、目标等方面的讨论。
+ 讨论了认知偏见、绩效因素、自我评价、同行评估、物理方面、外部影响和内部影响。
+ 强调了写作作为思考工具的重要性，以及如何通过书面表达来改进分析。
+ 提供了关于决策制定和行为偏见、概率和信念中的偏见的补充信息。

+ 情报问题陈述的发展
+ 第4章介绍了如何发展情报问题陈述，包括框架设定、指导方针、接收初始情报问题、评估问题、为框架问题进行研究、撰写情报问题陈述和核实情报问题陈述。
+ 讨论了在现实生活中发展情报问题陈述的方法，并提供了使用补充材料和示例的指导。
+ 提供了多种客户视角和目标视角的指南，以及针对外部问题、自发起的问题和特定区域问题的示例。

+ 分析计划的制定
+ 第5章讨论了如何从当前状态过渡到目标状态，构建分析计划，包括识别实质性分析领域、列出主要里程碑、列出必须完成的活动、识别其他所需资源、制定总体时间表和编写分析计划。
+ 强调了分析计划在指导分析过程中的重要性，并提供了总结。

+ 信息计划的开发
+ 第6章介绍了如何建立信息计划，包括确定广泛的信息获取主题、确定最佳的信息定位资源、选择信息存储和检索系统、确定谁将获取已识别的信息、创建信息获取时间表和编写信息计划。
+ 提供了信息来源和信息存储与检索系统的补充信息，并给出了信息计划的示例。

+ 信息获取的过程
+ 第7章讨论了信息获取的相关性和执行搜索策略的过程，包括制定初步搜索策略、执行搜索策略、确定搜索是否完成、获取和管理搜索过程中识别的相关材料。
+ 提供了布尔搜索的入门指南，包括OR逻辑、AND逻辑、NOT逻辑、组合AND和OR逻辑以及限制性AND变体。

+ 分析技术的应用
+ 第8章概述了分析技术的应用，虽然具体内容未给出，但提到了SATs（可能是结构化分析技术）。
+ 这一章节可能涵盖了各种分析方法和技术，用于处理和解释收集到的信息。
