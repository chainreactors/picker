在仓颉语言中，用于存储字符的数据类型分为字符类型（Rune）和字符串类型（String）。字符类型通常用于存储单个字符，而字符串类型则用于存储多个字符。

#### 使用字符类型存储单个字符

仓颉语言使用 `Rune` 表示单个字符。字符类型有三种表示方式：单字符、转义字符和通用字符。这些字符都使用一对单引号 `''` 包裹。

> 提示：在仓颉语言的当前版本中，字符类型仍然使用 `Char` 表示。不过，仓颉已经引入了 `Rune`，并且 `Rune` 与 `Char` 预期是相同的（即语法相同）。未来，`Char` 将被移除。因此，虽然书中示例使用了 `Char`，但官方文档已统一使用 `Rune`。为了与官方规范一致，本文后续也将使用 `Rune`。

##### 单个字符

单个字符指的是将某个 Unicode 字符（除反斜线 `\` 外）定义在一对单引号中。例如：

```
let rune1: Rune = 'X';  // 存储字符 'X'
let rune2: Rune = 'g';  // 存储字符 'g'
let rune3: Rune = '一';  // 存储汉字 '一'
let rune4: Rune = '仓';  // 存储汉字 '仓'
let rune5: Rune = '颉';  // 存储汉字 '颉'
```

##### 转义字符

转义字符以反斜线 `\` 开头。反斜线的作用是将其后的一个字符进行转义，用于表示某个具有特定含义的字符。例如换行符、制表符等，无法使用单个字符来表示，这时就可以使用转义字符来表示。常用的转义字符如下：

| 转义字符 | 含义 | 作用 |
| --- | --- | --- |
| `\n` | 换行符 | 将当前位置移到下一行的开头 |
| `\r` | 回车符 | 将当前位置移到本行的开头 |
| `\t` | 水平制表符 | 将当前位置移到下一个制表位 |
| `\\` | 反斜线 | 表示一个反斜线字符 |
| `\'` | 单引号 | 表示一个单引号字符 |
| `\"` | 双引号 | 表示一个双引号字符 |

以下是一些代码示例，展示了如何使用常用的转义字符：

```
let newline = '\n';  // 换行符
let tab = '\t';      // 水平制表符
let backslash = '\\';  // 反斜线字符
let singleQuote = '\'';  // 单引号字符
```

##### 通用字符

通用字符的单引号内以 `\u` 开头，后面加上一对花括号 `{}` 中的 1~8 个十六进制数，即可表示对应的 Unicode 值所代表的字符。例如，仓字的十六进制 Unicode 编码为 `4ed3`，字符 `仓` 可以表示为通用字符 `\u{4ed3}`。类似地，“颉” 字的 Unicode 编码为 `9889`，字符 `颉` 可以表示为通用字符 `\u{9889}`。

```
let rune6: Rune = '\u{4ed3}';  // 表示字符 '仓'
let rune7: Rune = '\u{9889}';  // 表示字符 '颉'
```

通用字符的优点是可以表示所有 Unicode 字符，缺点是因为各种字符的 Unicode 编码难以记忆，因此使用时需要查阅相关编码表。

#### 使用字符串类型存储多个字符

在仓颉语言中，字符串类型用于表示文本数据，可以包含多个有序的 Unicode 字符，使用 `String` 表示。字符串类型字面量有三种形式：单行字符串字面量、多行字符串字面量和多行原始字符串字面量。

##### 1. 单行字符串字面量

单行字符串字面量只能书写在一行，不允许跨越多行。它以一对双引号定义，双引号内可以是任意数量的 Unicode 字符（除了双引号和单独出现的 `\` 外）。字符串中可以使用转义字符。举例如下：

```
let str1: String = "";  // 空字符串，包含 0 个字符
let str2: String = "X";  // 只包含 1 个字符的 String
let str3: String = "你好仓颉";  // 包含多个字符的 String
let str4: String = "你好飞仓颉";  // 包含转义字符的 String
let str5: String = "\"你好仓颉\"";  // String 中的双引号必须使用转义字符
```

当读取单行字符串时，仓颉会从左到右将第一个双引号视为字符串开始的标志，双引号之后的字符就是字符串本身的内容，直到遇到第二个双引号为止，表示字符串结束。类似 `Rune` 类型的单引号字符，字符串的双引号边界的双引号仅用于标识，并不会输出。如果使用 `println` 函数输出字符串，结果如下：

```
println("你好仓颉");  // 输出：你好仓颉
```

需要注意的是，反斜线 `\` 不能单独出现在字符串中，必须用作转义字符。如果需要在字符串中表示反斜线，可以使用转义字符 `\\` 来表示。例如，在上例的 str5 中，双引号需要转义：

```
println(str5);  // 输出："你好仓颉"
```

##### 2. 多行字符串字面量

多行字符串字面量允许跨越多行，并且用三对双引号 `"""` 来定义。以三对双引号开始，紧接着是字符串的内容（可以跨越多行），最后同样以三对双引号结束。内容可以是任意数量的 Unicode 字符，并且可以跨越多行。举例如下：

```
let str1: String = """
 实践于林下河，
 仗剑踏朝日破。
 行侠快若轻胜马，
 谁惧？
 妄想种田平生。
 """
```

##### 3. 多行原始字符串字面量

多行原始字符串字面量用若干个井号 `#` 和一对双引号 `""` 定义。以一个或多个井号加上 1 个双引号开始，紧接着是字符串的内容，最后以相同数量的井号加上与开头相匹配的双引号结束。字符内容可以是任意数量的 Unicode 字符，并且可以跨越多行。举例如下：

```
let str1: String = #""实践于林下河""#;
let str2: String = ##""仗剑踏朝日破""##;
let str3: String = ###""行侠快若轻胜马，\n 谁惧？\n妄想种田平生。""###;
```

多行原始字符串字面量中的所有内容都会按原始格式保留（转义字符不会被转义），常常用于表示文件路径。对路径进行处理时，例如下面的文件路径 `C:\tools\WordProcessor.docx`，你可以写：

```
let path: String = r#"C:\tools\WordProcessor.docx"#;
```

##### 3.2.3 在输出字符串时使用换行符和制表符

在输出字符串时，可以使用换行符和制表符来组织输出的内容，使得输出结果更易阅读和美观。如果要输出的字符串内容较长，可以考虑在合适的位置使用换行符 `\n` 来换行。举例如下：

```
main() {
    var info: String = "杨柳青江水平，四郎江上唱歌声。\n东边日出西边雨，道是无晴却有晴。";
    println(info);
}
```

编译并执行以上代码，输出结果为：

```
杨柳青江水平，
四郎江上唱歌声。
东边日出西边雨，
道是无晴却有晴。
```

在输出以上内容时，我们可以进一步优化换行的代码，使得代码更清晰：

```
main() {
    var info: String;
    info = "杨柳青江水平，\n\t四郎江上唱歌声。\n\t东边日出西边雨，\n\t道是无晴却有晴。";
    println(info);
}
```

编译并执行此代码，输出结果为：

```
杨柳青江水平，
    四郎江上唱歌声。
    东边日出西边雨，
    道是无晴却有晴。
```

使用制表符 `\t` 可以很好地对齐输出的内容。例如，以下代码使用 `print` 函数输出了一些人名：

```
main() {
    print("韩愈");
    print("\t柳宗元");
    print("\t欧阳修");
    print("\t苏轼");
    print("\n");  // 换行
    print("\t苏辙");
    print("\t曾巩");
}
```

编译并执行以上代码，输出结果为：

```
韩愈    柳宗元    欧阳修    苏轼
    苏辙    曾巩
```

函数 `print` 与 `println` 的主要区别是，`print` 在输出内容后不会换行，而 `println` 在输出内容后会自动换行。

#### 拼接字符串

在仓颉语言中，可以使用操作符 `+` 对字符串进行拼接。例如：

```
println("东风夜放花千树，" + "更吹落，" + "星如雨。");
```

上述代码的输出结果为：

```
东风夜放花千树，更吹落，星如雨。
```

也可以使用复合赋值操作符 `+=` 来对字符串进行拼接。示例如下：

```
var str = "路漫漫其修远兮";
str += "\n吾将上下而求索";  // 等价于 str = str + "\n吾将上下而求索"
println(str);
```

输出结果为：

```
路漫漫其修远兮
吾将上下而求索
```

在拼接字符串时必须要注意操作数的数据类型。只有当左操作数和右操作数均为 `String` 类型时，才可以使用 `+` 或 `+=` 进行拼接。否则，会导致编译错误。例如：

```
println('x' + "yz");  // 编译错误：左操作数为 Rune 类型，右操作数为 String 类型
println("xy" + 'z');  // 编译错误：左操作数为 String 类型，右操作数为 Rune 类型
println(10 + "个苹果");  // 正确：左操作数转换为 String 类型后拼接
```

如果需要将其他类型的数据与 `String` 类型拼接，可以先调用 `toString` 函数将其转换为 `String` 类型，然后再通过 `+` 或 `+=` 进行拼接。示例如下：

```
main() {
    var side = 3;
    var info: String;
    info = "边长为" + side.toString() + "的正方形的面积为" + (side ** 2).toString();
    println(info);
}
```

编译并执行以上代码，输出结果为：

```
边长为3的正方形的面积为9
```

虽然使用以上方式可以实现其他类型与字符串的拼接，但在实际操作中，推荐使用插值字符串来实现更加简洁且可读性更好的代码。

#### 使用插值字符串

插值字符串用于向字符串面量插入不同类型的表达式。通过将表达式插入字符串中，以有效避免字符串拼接时的类型转换问题。插值字符串的语法格式如下：

```
"字符串${表达式}"
```

插值字符串中的每个插值表达式的值都会被计算出来，最终得到的仍是一个字符串。插值字符串中可以包含简单的表达式，也可以包含一个或多个声明或表达式，中间以分号隔开。每个插值表达式中的值会自左至右依次执行。示例如下：

```
main() {
    var info: String;
    var side = 3;
    info = "边长为${side}的正方形的面积为${side ** 2}";
    println(info);
}
```

编译并执行以上代码，输出结果与前面的拼接结果相同，但避免了类型转换的复杂性。

```
边长为3的正方形的面积为9
```

插值字符串除了包含简单的表达式，还可以包含复杂的逻辑操作。例如：

```
main() {
    var info: String;
    info = "笔记本价格为：${let price = 150; price}元";
    println(info);
}
```

输出结果为：

```
笔记本价格为：150元
```

#### 元组类型

元组类型用于将两个或多个不同类型的值合在一起，形成一个新的类型。元组类型的长度是固定的，定义后不能再被扩展或缩减。元组的表示方式如下：

```
(T1, T2, ..., TN)
```

元组的长度是固定的，在定义元组后，其内容不能再修改。例如，定义一个三元组 `weather`：

```
var weather: (String, String, Int8) = ("周一", "晴", 19);
```

如果需要访问元组中的元素，可以使用索引值访问，索引从 0 开始。例如：

```
println("天气: ${weather[1]}，气温: ${weather[2]}度");
```

输出结果为：

```
天气: 晴，气温: 19度
```

#### Unit 和 Nothing 类型

仓颉中的某些表达式的类型是 `Unit` 或 `Nothing`。其中 `Unit` 表示返回值为空的类型。Nothing 通常表示永远不会成功完成的表达式（例如抛出异常的表达式）。常用的 Unit 类型可以如下定义：

```
main() {
    var i = 0;
    i++;
    println(i);
}
```

编译并执行以上代码，输出结果为：

```
1
```

在这个例子中，`i++` 表示递增操作，虽然它有副作用，但返回值为 `Unit`，表示该操作没有可用于进一步操作的返回值。
