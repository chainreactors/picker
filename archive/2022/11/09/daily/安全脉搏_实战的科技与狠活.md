## 0x01前言

在一次攻防中遇到了一个系统，过程平平无奇但是也费了些时间，这里简单的记录下。

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882908.png)

## 0x02目标

这次的目标看着就是平平无奇

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882914.png)

我一勺～(不是) 一手弱口令直接进入后台,在后台一处功能点存在文件上传

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882922.png "null")

## 0x03科技

马不停蹄开始冲刺

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882925.png)

内容为图片，显然有waf检测后缀，没啥好办法一个个进行测试过程就直接省略了。结论：在"filename"最后位置加一个“ " ”直接就可以绕过。

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882942.png)

既然存在waf检测后缀那肯定也存在内容检测，直接上免杀马，就在自我感觉良好的时候免杀马分分钟被烂了。

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882945.png)

这样那就只能和后缀一样一点一点进行测试，这里使用二分法测试下到底是哪儿触发了waf的规则被拦截然后再想办法进行绕过。经过测试有三个点触发waf规则 1、：<%后面跟任何字母都不行(<%try-------)

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882947.png "null")

2、：`<%@ Page Language="CS" %>` 有它就不行猜测也是和1类似。3、：中间代码位置`new System.IO.MemoryStream(); object o.....`

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882949.png "null")

1和3可以使用同一种方式绕过:利用注释添加脏字符

```
/*asdasd--------asdas*/
```

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882950.png)

2可以使用另一种方式进行替换

```
 <script            language=csharp runat=server>
```

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882957.jpeg)

成了！

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882959.png)

## 0x04狠活

说是狠活其实也就是巧合 执行命令"whoami" iis权限 再执行其他命令然后发现已经没一点反应了，但是shell还在 emmmm

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882960.png "null")

然后发现哥斯拉自带的“sweetpotato”可以提权并执行命令，但是只能执行一个然后就卡死，所以需要一个简单高效的命令一次搞定（todesk），上传todesk-->sweet potato执行安装-->文件管理查看配置文件-->成功连接。

![](https://secpulseoss.oss-cn-shanghai.aliyuncs.com/wp-content/uploads/1970/01/beepress-image-190585-1667882961.png)

总算搞定。
