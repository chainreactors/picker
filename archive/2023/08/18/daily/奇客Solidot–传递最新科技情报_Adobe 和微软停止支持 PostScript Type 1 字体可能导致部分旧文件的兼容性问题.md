## Adobe 和微软停止支持 PostScript Type 1 字体可能导致部分旧文件的兼容性问题

[![软件](https://icon.solidot.org/images/topics/topicsoftware.png?123)](/search?tid=15 "软件")

[Wilson](/~Wilson) (42865)发表于 2023年08月17日 14时37分 星期四 [新浪微博分享](//service.weibo.com/share/share.php?url=//www.solidot.org/story?sid=75821&appkey=1370085986&title=Adobe%20%E5%92%8C%E5%BE%AE%E8%BD%AF%E5%81%9C%E6%AD%A2%E6%94%AF%E6%8C%81%20PostScript%20Type%201%20%E5%AD%97%E4%BD%93%E5%8F%AF%E8%83%BD%E5%AF%BC%E8%87%B4%E9%83%A8%E5%88%86%E6%97%A7%E6%96%87%E4%BB%B6%E7%9A%84%E5%85%BC%E5%AE%B9%E6%80%A7%E9%97%AE%E9%A2%98 "新浪微博分享")
![](https://icon.solidot.org/images/a7c7.png)

**来自神间失格**

Adobe 和微软 Office 应用今年先后宣布停止支持 PostScript Type 1 字体。PostScript 是 Adobe 在 1980 年代为专业数字排版开发的大纲字体规范编码的字体文件。PostScript 让文字和图像的自由缩放、旋转和移动成为可能，今天大部分打印机都依赖于 PostScript 或其后续技术 PDF。PostScript 最重要的字体就是 Type 1。为了避免 Adobe 一家独大，苹果在 1990 年代创建 TrueType，并将其授权给微软。Adobe 和微软后来合作开发了 OpenType 字体格式去取代 TrueType 和 PostScript Type 1。MS Office 的 Windows 版本早在 2013 年就停止支持 Type 1，本周它的 Mac 版本也终止了对 Type 1 字体的支持。此举有可能对旧的文件造成兼容性问题，在新版本 Office 应用中打开旧文件可能会显示字体缺失的错误信息，而应用在尝试替代 Type 1 字体时也可能导致布局问题。
