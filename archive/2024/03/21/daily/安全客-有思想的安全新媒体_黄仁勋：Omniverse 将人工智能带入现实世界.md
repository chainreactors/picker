## NVIDIA 创始人兼首席执行官黄仁勋近日在 GTC 会议上宣布NVIDIA Omniverse Cloud将以 API 形式提供。

## Omniverse 将人工智能带入现实世界

黄说，下一波人工智能浪潮将是人工智能学习物理世界。

“我们需要一个模拟引擎，为机器人以数字方式呈现世界，这样机器人就有了一个可以去学习如何成为机器人的健身房，”他说。 “我们称这个虚拟世界为 Omniverse。”

因此，NVIDIA 宣布[NVIDIA Omniverse Cloud](https://www.nvidia.com/en-us/omniverse/)将以 API 形式提供，从而扩展了世界领先平台的覆盖范围，以便在整个软件制造商生态系统中创建工业数字孪生应用程序和工作流程。

五个新的 Omniverse Cloud 应用程序编程接口使开发人员能够轻松地将核心 Omniverse 技术直接集成到数字孪生的现有设计和自动化软件应用程序中，或者用于测试和验证机器人或自动驾驶车辆等自主机器的模拟工作流程中。

为了展示其工作原理，黄分享了一个机器人仓库的演示——使用多摄像头感知和跟踪——监视工人并协调机器人叉车，这些叉车在整个机器人堆栈运行的情况下自动驾驶。

黄仁勋还宣布，NVIDIA 正在将 Omniverse 引入 Apple Vision Pro，新的 Omniverse Cloud API 可以让开发人员将交互式工业数字孪生流传输到 VR 耳机中。

![]()

全球一些最大的工业软件制造商正在采用 Omniverse Cloud API，其中包括 Ansys、Cadence、Dassault Systèmes 的 3DEXCITE 品牌、Hexagon、微软、罗克韦尔自动化、西门子和 Trimble。

NVIDIA Omniverse 平台由 API、SDK 和服务组成，可帮助开发人员集成 RTX 渲染技术和通用场景描述 (OpenUSD)融入他们现有的软件工具和模拟工作流程。 Omniverse SDK 允许用户使用低代码和无代码示例应用程序以及易于修改的扩展从头开始构建工具和工作流程。现在，Omniverse Cloud API 使用户能够通过 OpenUSD、RTX、加速计算和生成 AI 技术增强其应用程序和工作流程。此外，Omniverse 允许在任何支持 RTX 的工作站或虚拟工作站上部署自定义应用程序。它还可以从 Omniverse Cloud 平台即服务托管和流式传输应用程序。

新的 Omniverse Cloud API 可以集体或独立使用，包括：USD Render（生成完全光线追踪的 OpenUSD 数据 NVIDIA RTX™ 渲染）、USD Write（让用户与 OpenUSD 数据交互并修改其 OpenUSD 数据）、USD Query（用于构建场景查询和交互式场景、更新和美元变化跟踪器USD Notify和Omniverse Channel，这是一个支持跨用户、工具和世界协作的工具。

NVIDIA 首席执行官黄仁勋展示了西门子业界领先的基于云的产品生命周期管理 (PLM) 软件 Teamcenter X。 Omniverse Cloud API 增强了 Teamcenter X，能够将设计数据连接到 NVIDIA 生成式 AI API，并在 Teamcenter X 中使用 NVIDIA RTX 渲染。除了西门子之外，许多重工业领导者也利用 Omniverse Cloud API 来解决多个用例。例如，ANSYS正在采用 Omniverse API 在其自动驾驶汽车和 6G 仿真解决方案中实现数据互操作性和 RTX 可视化。同样，计算软件提供商节奏Omniverse Cloud API 正在其 Cadence Reality 数字孪生平台中采用，以便在构建数字孪生数据中心之前对其进行设计、仿真和优化。 Omniverse Cloud API 还将仿真工具和应用程序的生态系统与行业领先的传感器解决方案提供商连接起来，以使用基于物理的传感器仿真来实现全栈训练和测试应用程序。
