48 岁的微软，再次站上科技潮头。

**作者 | 宛辰 郑玥**

经历过「Win95」时期的人们，可能还记得一个小东西——Office 97 套件中的「大眼夹」Clippy，一个设计搞笑、主要卖萌，偶尔给做文档的社畜提出点修改意见的虚拟形象。

时隔 25 年后，一个用 AI 技术重构升级版的「大眼夹」复活了——改名叫 Copilot 的它，没有变秃，但是更强了。

美国时间 3 月 16 日早上 8 点，微软召开发布会，正式曝光由最新 AI 技术 GPT-4 加持的 Microsoft 365。由微软 CEO 萨提亚·纳德拉领衔的高管队伍，用一个个实例告诉人们，背靠 GPT-4 技术，微软的「商务全家桶」软件，从 Word 到 Excel 再到 PPT，真的已经「成熟」到可以自己写文件，做表格了。

**「Copilot，全新的工作方式。」**

微软当天活动的副标题，醒目地提示人们，就像当年 Office 套件在信息化浪潮中改变了办公室的工作方式一样；活用了 AI 技术的 Copilot，从今天开始就将彻底改变互联网时代人们的工作方式。

在无所不能的 Copilot 背后，微软在 AI 时代「新基建」的真正野心，也通过此次发布会变成「明牌」。

**01**

**365，天天「GPT」**

美国时间 3 月 16 日早上 8 点，微软举办了一场名为「用人工智能重塑生产力」的发布会，**官宣将旗下商业软件 Microsoft 365 全线接入 GPT-4，并推出一款新 Copilot，旨在帮助人们生成文档、电子邮件、演示文稿等。**微软去年发布的 Copilot 时，它还是一个使用 OpenAI 的技术帮助程序员自动编写计算机代码的工具。

「Copilot 是一种全新的工作方式。」Microsoft 365 负责人 Jared Spataro 说，「它与你一起工作，嵌入到数百万人每天使用的程序中：Word、Excel、PowerPoint、Outlook、Teams 等等」

新的 Copilot 在 Microsoft 365 界面中和其他应用程序并排，作为聊天机器人出现在侧边栏，用户可在 Microsoft 的 Office 软件中使用 Copilot，包括 Word、 PowerPoint、Excel、Outlook 和 Teams。

**Copilot 本质是聊天机器人，用户可以通过聊天对话，让 Copilot 帮助你进行工作生产：**

**·** Word 中的 Copilot ：与人们一起工作，进行写作、编辑、总结和创作。

**·**PowerPoint 中的 Copilot ：通过自然语言命令将想法转化为设计好的演示文稿。

**·**Excel 中的 Copilot ：帮助用户在短时间内进行洞察、识别走势，创建具有专业外观的数据可视化内容。

**·**Outlook 中的 Copilot ：可以帮助综合管理邮箱，让用户腾出时间花在实际通信上。

**·**Teams 中的 Copilot ：直接在会议对话的上下文中提供实时摘要以及操作项，从而提高会议效率。

**·**Power Platform 中的 Copilot ：将通过在 Power Apps 和 Power Virtual Agents 中引入两项新功能，帮助所有技能水平的开发人员使用低代码工具，加速和简化开发。

**·**Business Chat 中的 Copilot ：汇集了来自文档、演示文稿、电子邮件、日历、笔记和联系人的数据，以帮助总结聊天、撰写电子邮件、查找关键日期，还能根据其他项目文件编写计划。

在 Word 中，Copilot 可以根据指令起草文档，使用者可以在其生成文本之上自由编辑。有 Word 文档，就可以直接基于文档创建 10 页 PPT。Excel 当然也可以，甚至可以在给出数据后，即时来一个 SWOT 分析，直接生成数据透视表。

在会议功能中，Copilot 可以在会前为用户提供即将举行会议的信息，会中转录 Microsoft Teams 的会议，会后可以总结整个会议要点。以及进行会议提醒、事项提醒，为用户准备项目更新、组织变化（例如最近的招聘），甚至是你休假回来后，帮你更新同事们的进展。

Copilot 也应用于 Outlook 邮箱功能中，帮助分辨信息让用户花更少的时间处理邮件，总结电子邮件线程，甚至可以写带有草稿的回复，以便用户调整语气、邮件长度等。这与此前爆料接近，The Information 在今年 1 月的一篇报道中写道，一位接近人士透露，微软讨论了使用 OpenAI 的软件在 Word 和 Outlook 中创建聊天机器人风格的程序，根据提示编写整段文本。

![](https://mmbiz.qpic.cn/mmbiz_gif/8cu01Kavc5ZDhe4gcyWB8TcwdSsuyuDhULAWicJcQdHYOuSjQVXDz3tOtvibcClHNaynLVXR0BwHeC37eibnmzyxQ/640?wx_fmt=gif)

Copilot 在 Outlook 中帮助用户撰写邮件并抄送对的同事 ｜图片来源：微软

Spataro 说到， Copilot 不止是「Microsoft 365 Copilot」，而是一个系统。**Copilot 系统将  GPT-4 结合到 Word、Excel 和 PowerPoint 等 Microsoft 365 应用程序以外，还结合到 Microsoft Graph（连接多种服务和设备的 Microsoft API 开发人员平台）中**。

基于 Microsoft Graph，微软还计划推出适用于所有 Microsoft 365 的办公聊天功能 Business Chat。Microsoft Graph 可以将文档、演示文稿、电子邮件、笔记和联系人汇集到 Microsoft Teams 中的单个聊天界面中，在其中生成摘要、计划概述等。

微软当日发文表示，目前正在与 20 位客户一起测试 Microsoft 365 Copilot，并将在未来几个月内扩展预览版，「我们将在未来几个月分享更多关于定价和细节的信息。」

模型就有的问题，应用也逃不掉。Copilot 和 GPT 一样，存在准确真实问题，不能完全被信任。Spataro 在今天直播中说到，**「有时 Copilot 会做对，有时它会出错，但可以给你一个不完美的想法，让你领先一步」**。

**Copilot 可能是一个不甚准确但很好用的办公助理**——这不禁让人担心，Copilot 在办公场景下可能会错漏百出，不需要担心的是，它仍然会被大规模快速应用。

微软内部和外部的一些人都对人工智能软件的发展如此迅速感到担忧，但 Spataro 并不太担心，他在发布会说到，「为了满足客户的需求，我们必须迅速而负责任地采取行动，边走边学。我们正在与一小部分客户一起测试 Copilot，以获得反馈并改进我们的模型，我们将很快就能改进。」

其实早在今年 1 月，微软就被爆料计划将这些产品接入 GPT-4。据 The Information 1 月 7 日报道，一位接近人士透露，微软工程师们正在开发在客户数据上训练这些模型，而不会泄露给其他客户或落入不良行为者手中。如今工具已经发布，但这个问题的解决程度并未在发布会中着墨解释更多。

**02**

**新 AI 「基建战」**

一个月前，前去硅谷湾区走访调研的两位投资人均表示，哪怕硅谷也没有很好地接入大语言模型的 ToB 场景，还有很多问题有待解决，比如 B 端场景数据与如何与大模型融合，交互怎么做……

看完微软发布会后，一位投资人向极客公园感叹：「只能说，人家的工程化能力太强了。」

的确，微软将大语言模型与 Office 办公场景的落地难点涉及很多工程问题。

出门问问创始人李志飞解释道：把这个体验做出来不是随便接入 GPT 接口，而是一个系统性工程（见下图）。**喜欢胡说八道的 LLM 接在 Microsoft Graph（也就是你的数据）上，终于接地（所谓 grounding）了，然后给 apps 提供一本正经的自然语言服务——这给大家提供了 LLM 怎么跟具体数据和应用场景结合的完美示范**。

![](https://mmbiz.qpic.cn/mmbiz_png/8cu01Kavc5ZDhe4gcyWB8TcwdSsuyuDhNm9iaqk3ARsia6zAiaOriaewRAxQAwg02McOeYwyIk8qaiagEnYwVO8D7Yg/640?wx_fmt=png)

Microsoft 365 Copilot 系统｜图片来源：微软

无论是接入 GPT-4 的 New Bing，还是可以结合个人数据的 Copilot AI 助手，微软显然是有预谋的，它还在下更大的棋。

除了搜索入口和企业办公这两个最广泛的场景，**微软还提早布局了承载大语言模型的基础设施——微软云 Azure**。

这是因为，训练 GPT-4 这样的大语言模型，需要依赖强大的算力，微软了解 OpenAI 需要什么样的超级计算基础设施以及所需的规模，因此，**团队在几年前就基于 GPT 大模型的特性，对 Azure 的架构进行了调整**。

微软在自己官博也解释了团队押注 Azure、开启人工智能革命的过程。微软 Azure 高性能计算（HPC）产品负责人 Nidhi Chappell 表示，「模型越大，拥有的数据越多，训练时间越长，模型的准确性就越好，」「因此，**团队肯定有强烈的推动力让更大的模型，接受更长时间的训练，这意味着你不仅需要拥有最大的基础设施，还必须能够长时间可靠地运行它**。」

并且，大模型迭代所需要的算力还在继续扩大，根据此前报道，OpenAI 训练大模型所需要的算力每三四个月翻番。

**大模型越强，Azure 越强**。DCM 董事合伙人曾振宇解释道，由于 GPT 大模型从第一天就是在 Azure 这样的云资源上搭起来的，所以原生性很好。同时，如果 GPT 模型变得更大，Azure 的用量自然会迅速扩大，而且基于 GPT 上面派生出来的其他应用，天然会跟 Azure 的亲密度或者契合度更高，进一步增加 Azure 的用量。

**随着公有云的竞争日趋白热化，大模型开启了差异化的新故事**。Azure 和 Google Cloud 同年推出，但显然，Azure 现在已甩开后者两个身位。

事实上，随着 OpenAI 和微软都试图迅速扩大其业务，**Azure 现有的算力已经出现紧张了**。据 The Information 报道，内部员工使用需要申请配给，客户使用也需要排队。微软正在为 Office、Bing 中的 OpenAI 产品保留许多人工智能芯片，微软自己的研究已经让位于人工智能模型开发。

两个月前，当微软被爆百亿美元加注 OpenAI 时，大家还在讨论值不值得。现在，拾象科技李广密认为，微软其实是给自己找了个大客户，投出去的钱拿了 Open AI 股份，又通过 Azure 赚回来，公司股价还能拉升，Open AI 也拿到了大把资源，植入 Bing、Office 最佳入口，占据了最佳生态位，还可以通过 CVC 投资赚钱，还把 AI 热潮推向一个高峰，简直是一举「数赢」。

估计谁也没能想到，错过了整个移动时代，但是在云计算时代翻身赶上的微软，在成军 48 年之后，在 AI 时代，重新成为硅谷最热门的公司，一个飞向天边的「大象」。
