## SAP User Experience

与上一版本一样，在 2302版本中，协作也是头等大事。Collaborative Cloud ERP的首个功能为企业员工间协作的方式添加了“作为聊天共享”这一选项。作为 Microsoft Teams 集成一部分，现已全面推出。在sven.denecken的[2208版本亮点](https://blogs.sap.com/2022/08/01/highlights-of-the-sap-s-4hana-cloud-2208-release/)的博文中，他介绍了此功能如何通过支持您将基于 SAP Fiori 元素应用的内容直接共享给 Microsoft Teams，从而促进所有 SAP S/4HANA Cloud, public edition用户之间的协作。

此外，我们还通过 Early Adopter Care 计划推出了“Share as Tab”作为选项卡分享选项。此功能允许您实时通过 Microsoft Teams Channel 以标签形式与同事共享业务数据，从而使协作更加容易！更具可视性地轻松访问最新的信息，您的同事可以即时处理给定任务并做出明智的决策。在此处[注册](https://influence.sap.com/sap/ino/#campaign/3051)以抢先体验测试此功能，并快人一步提供您的反馈！

## 财务

2302版本在财务领域，SAP向您提供了项目制造的按订单设计 （ETO）的功能。借助 ETO 解决方案，您可以：查看项目的基于事件的生产成本过账（成本包括间接费用、差异、在制品等），执行基于事件的收入确认，运行利润分析以分析项目获利能力。新的范围项目 6GD首先发布在德国和美国的国家版本下。这个端到端流程从公司收到新产品规格的销售订单开始，连接了项目及其后续流程的必要步骤，例如销售、采购和制造，保证了项目管理和成本控制的成功进行。

另一个值得一提的更新是关于所有会计核算活动的核心——分类账。如果您的公司在多个国家运营，则可能需要根据不同的会计标准（即使用并行会计）提供财务报表。在 SAP S/4HANA Cloud 2302，我们现在支持最多三个并行标准分类账，以满足同时使用一个本地和两个公司会计准则（例如 IFRS 和 US-GAAP）的业务需求。您可以通过系统的“Scope”阶段选择分类账场景。

想要了解财务在2302版本的更多亮点更新，您可以查看专家bob.shen。

## 寻源与采购

在2302版本中，SAP在采购模块发布了一系列创新，以期提高您的管理效率。例如，在采购订单管理的创新——我们为采购订单提供催款提醒，以支持采购员创建货物或服务交付提醒并发送给其业务伙伴，以提醒他们逾期的交货日期。采购员筛选适用于催款提醒的采购订单，并且可以打印或通过电子邮件发送催款或逾期信息。您还可以使用设置参数，以及使用应用计划采购订单的催款提醒输出，定期生成计划此类输出。​通过增强采购商和供应商之间的交互，旨在确保及时交付货物或及时履行服务。同时最大限度地减少这些任务附带的手动工作量。新的自助采购应用为员工或临时用户提供整合的一站式解决方案，从而提供快速、直观和高效的购物体验。

## 销售

在2302版本的销售相关业务线中，SAP发布了一系列的功能增强，其中一个亮点在于，您现在可以根据角色配置对于销售订单中价格和成本详细信息的访问权限。也就是说，您可以配置销售订单中，不同的用户界面上所显示的价格元素区域中的价格信息。系统将根据您的访问级别配置以及已分配业务目录销售 - 销售订单处理 (SAP\_SD\_BC\_SO\_PROC\_MC) 的角色的设置来控制访问。因此，您可以灵活地限制对敏感价格和成本信息的访问，并控制销售订单流程中的更改功能，例如，销售经理可以查看所有价格条件类型并且修改，销售专员可以查看不能修改，而后勤仓库专员不能查看内部成本和利润率价格条件类型。

## EPPM

在2302版本中，项目控制 – 资源规划和采购 (‏4RF‏)，新增了一个需求类型：项目库存，临时采购。该类型允许你计划和采购长周期项目，触发计划独立需求。运行MRP, 通过生产或者采购订单满足该需求。使用项目类型“投资项目”，“有收入的项目”可以使用该项目需求-临时采购。

另外，2302版本在ETO的业务场景支持上有了进一步的支持。现在您可以在项目成本报告中显示生产订单成本。通过这种方式，项目财务总监可以深入了解生产订单中记录的实时成本。

针对以上两点更新的具体内容，您可以查看fengchen。

## 混合云 2层架构

总部和子公司之间的主数据复制是双层 ERP场景能平稳运行的最关键方面。现在，您可以使用 SAP MDI 服务（SAP BTP 上的多租户云服务），提供跨混合架构的主数据的一致视图。SAP MDI 服务基于 SAP One Domain Model，其中可以将对象映射到使用不同语言的应用程序的中央域，并且可以调整配置和事务数据。MDI 使用中心和轮辐模型实现主数据集成的标准化，并支持客户特定字段的可扩展性。因此，它为集成和扩展场景奠定了基础。欲知更多，请参看我们近期发布的MDI博客。

更多混合云（Two-Tier ERP）增强的内容可以查看 joey.li1
