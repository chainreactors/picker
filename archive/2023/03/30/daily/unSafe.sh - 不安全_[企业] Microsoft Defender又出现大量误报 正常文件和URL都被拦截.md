据 Microsoft 365 Status 发布的公告，企业 / 组织使用的 Microsoft Defender 又翻车了，大量正常文件和 URL 地址被误报，管理员收到海量的拦截通知。

[![[企业] Microsoft Defender又出现大量误报 正常文件和URL都被拦截](https://img.lancdn.com/landian/2022/03/93056.png)](https://img.lancdn.com/landian/2022/03/93056.png)

如果只是简单的误报可能还没什么，但误报对企业业务造成了混乱：有企业反馈称，从企业内部对外发送的电子邮件表面上是发出去了，但客户并没有收到。排查日志才发现，邮件中包含了链接，链接被 Microsoft Defender 误报，结果邮件直接被拦截了。

据说这个问题并不是今天才出现的，应该有好几天了，只不过现在微软才开始介入调查，微软表示正在检查服务遥测信息找出发生错误的根本原因并进行修复。

Microsoft Defender 出现误报是个比较频繁的事情，[之前还多次出现过把谷歌浏览器的安装程序给拦截了的](https://www.landiannews.com/archives/84737.html)，[还有一次把微软自己的 Office 给拦截了](https://www.landiannews.com/archives/93056.html)。

而且涉及的也不仅仅是企业，Windows 10/11 自带的 Microsoft Defender 有时候也可能出现误报，把部分正常软件给拦截。

这次出现的问题仅影响企业，企业 IT 管理员可以在 Microsoft 365 管理中心检查编号为 DZ534539 的跟踪报告。
