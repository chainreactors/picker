JavaScript 创始人布兰登艾奇创立的 Brave 浏览器在 2021 年推出了自己的搜索引擎，即 Brave Search，其中一部分搜索结果是调用微软 Bing API 的，也就是从必应搜索结果里获取内容填充到 Brave Search 里。

接下来的几个月 Brave 与微软签订的合同到期，为此 Brave 决定停止继续与微软合作，专注于自己的搜索爬虫开发，避免像 DuckDuckGo 那样受制于微软。

Brave 担心什么呢？主要有两个，第一是 Bing API 潜在的隐私问题；第二是微软以 Bing Chat 为借口，[Bing API 价格出现大幅度上涨](https://www.landiannews.com/archives/97357.html)。

[![Brave浏览器/搜索引擎宣布放弃调用必应搜索结果 担心微软瞎搞](https://img.lancdn.com/landian/2023/04/98466.png)](https://img.lancdn.com/landian/2023/04/98466.png)

**Brave 在博客中表示：**

*我们担心 Bing 服务的连续性，结果证明这是一个有先见之明的担忧，因为微软最近宣布其 API 价格大幅度上涨，这给部分依赖或完全依赖 Bing Search API 的搜索引擎带来了空前的压力。*

*在合同到期后这些依赖 Bing API 的搜索引擎将会证明依赖微软的后果。*

此前标榜隐私不追踪的 DuckDuckGo 被发现会将部分数据传输给微软，因为 DuckDuckGo 完全依赖于 Bing API，所以不得不与微软签订相关的协议。被发现后这引起了极大的争议，后来 DuckDuckGo 解决了隐私问题，不知道向微软付了多少补偿费。

依赖 Bing API 提供搜索服务的搜索引擎并不少，正如 Brave 所说，等合同到期就知道依赖微软有多惨了，而这也会证明 Brave 之前开发自己的搜索爬虫并逐渐降低 Bing API 使用确实是先见之明，目前 Brave Search 的搜索结果里只有 7% 来自 Bing API，因此合同到期后剔除这部分内容也不会有太大影响，后续 Brave 自己的爬虫会继续补上。
