具透 | 三星 One UI 5 评测：当 Android 13 遇上 iOS 16

对国内 Android 用户而言，三星的 One UI 在一众 Android 定制系统中一直以来也都算是 Others 一般的存在（就像它们的硬件一样）。

![boxcnCSJ6UHn5lXtiQERc4zwhld](https://cdn.sspai.com/editor/u_/cdf3ac5b34tf3i7qq87g?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

若论「本地化」，国行三星至今仅适配过在国内影响力十分有限的统一推送，日常使用连最基本的消息推送都很难保证；但论系统功能，因为三星近年与 Google 关系密切，加上 One UI 面世后三星对系统升级和软件体验的格外重视，One UI 对最新的原生 Android 特性跟进得却也最为及时，并且各种小功能也多到让人直呼「需要一本说明书」。

在本月中旬的三星 SDC 22 开发者大会中，三星正式官宣了 One UI 5，海外版 Galaxy S22 系列机型近期陆续开始收到了更新推送，国行版的内部测试也已经接近尾声。作为 Android 13 定制系统中适配速度相对较快的大厂，三星这次在 One UI 5 中又带来了哪些新东西？

**注：本文基于 Galaxy S22 Ultra 国行内测版本 S9080ZCU2ZVJC，国行版推送的最终版本功能体验或许会存在差异。**

**关联阅读：**

* [升级也要图个明白：8000 字详解 Android 13 正式版的 9 个新变化](https://sspai.com/post/75272)
* [具透 | 三星如何玩转 Android 12？One UI 4 上手体验](https://sspai.com/post/71136)

## One UI 总能用上新特性

对原生特性的及时跟进和合理保留，是 One UI 相比大部分定制 Android 系统的优势所在。所以我们可以在 One UI 5 中看到和体验到很多此前只存在于 Google Pixel 这类机型上的新东西。

**但问题在于，今年 Google 为 Android 13 准备的新特性本就不多**。我们自然也无法在这方面期待 One UI 5 能够拿出太多不一样的新东西。

比如 One UI 4 中适配原生 Android 12 的 Material You 动态主题的「调色板」，在 One UI 5 中也和原生 Android 13 一样，拥有了最多 16 种不同风格的壁纸颜色方案、8 种基本颜色方案和 4 种基本颜色撞色方案。

![](https://cdn.sspai.com/2022/10/30/9f0bd6342f234ae8709fb6f54d709e1b.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/a4d368ddbc3e5a252026b53d19adf3e7.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

同时，调色板的颜色方案同样也可以应用到第三方 app 图标了，已经适配的应用在开启这一功能后，几乎能够获得和 Google Pixel 相近的主屏幕观感——唯一的不同在于 One UI 5 在色彩选取上还欠缺 Google Pixel 那种功力，同样一张壁纸、选择同样的取色策略、都开启主题图标的情况下，One UI 5 的配色方案会显得更加厚重（上图 A1 列），在壁纸上的辨识度甚至因此变得更低了；Pixel 的色彩选择方案（上图 A2 列）则能同时保证图标观感和可视化需求。

![](https://cdn.sspai.com/2022/10/30/d7e19bb12cb80633f42664b92611bcc4.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/ef1d52a4bdae49f9bf2786441741c870.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

除了图标和配色，One UI 5 肉眼可见的视觉样式更新并不多，比如通知布局的变化对大多数用户来说都是难以察觉的。

![boxcnHFGF0Ju7xvyVbq4XECVn8e](https://cdn.sspai.com/editor/u_/cdf3ae5b34tf3aqo2o70?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

通知布局调整 | 图：三星

**好在 Android 13 最为核心的关键性功能更新得到了保留**。

比如「系统设置 - 常规管理」中新增了「应用程序语言」一项，可以让我们很方便地为应用设置不同于操作系统的其他语言。并且和原生 Android 13 的正式版一样，只有那些为应用配置了语言文件和清单声明的应用，才会出现在这个设置页面当中。如果你想设置的应用并未适配，也可以按照我们此前分享过的[方法](https://sspai.com/post/75272)通过 ADB 手动解除上述限制。

![](https://cdn.sspai.com/2022/10/30/48b41d5c27d28e049b308bd001dda618.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/4537ee0c7ca082c672742dff435b2e5c.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

**One UI 5 完整保留了 Android 13 正式版的运行时通知权限授予机制**。

在 Android 12 或更早的系统版本中，通知权限安装时默认授予，流氓应用可以肆无忌惮地发送通知来骚扰用户甚至借此实现「保活」，因此不仅是 One UI 5，所有 Android 定制系统其实都是有必要对 Android 13 的运行时通知权限进行适配和接入的。接入后，未适配 Android 13 的 app 将在初次运行时弹出通知授权窗口，适配了 Android 13 的 app 则可以根据实际情况选择授权弹窗时机。

![boxcnVrEbBWZzn029GWalqccbjd](https://cdn.sspai.com/editor/u_/cdf3af5b34tf2bu61ggg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

运行时通知授权弹窗

**Google 在 Pixel 机型上主推的一站式「安全中心」检查面板，在 One UI 5 中同样也有对应的产品——「隐私与安全」面板**。比较在意设备安全性的用户可以在这个面板中随时获取全面的设备安全状态报告，并且通过顶部的建议一键跳转解决遇到的安全风险。

![](https://cdn.sspai.com/2022/10/30/5b29564d4348651c6ef8cc061604b6a1.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/eda1741656a784c413fcbd1c0de52b35.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

**最后，Android 13 正式版的前台服务管理器（FGS）在 One UI 5 中也得到了保留**。有意思的是虽然三星依然稳坐 [Don't kill my apps](https://dontkillmyapp.com/) 杀后台排行榜第一，这项功能的入口在 One UI 5 中却有着更加合理的入口设置（位于多任务界面上方）和功能命名——运行前台服务的主要目的一般是保证后台活动，所以三星这里直接以用户视角将 FGS 改名叫做后台活动管理，对用户而言其实是更加通俗也更为直观的。

![boxcnFGi2q44xqB93lkT0Fuypig](https://cdn.sspai.com/editor/u_/cdf3ag5b34tf3aqo2o7g?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

## 三星也总会惦记着老对手

正如上文所说，Android 13 正式版因为更像是对 Android 12 的「补全」，实际能够为各大厂商们带去的新东西并不算太多。所以除了新系统的特性跟进，其他地方才是更加考验各大定制系统产品经理的环节。

如果你是三星 One UI 的产品经理，你会怎么做？此时的三星看向了自己的老对手 Apple。从某个测试版本开始，One UI 5 加入了大量我们在 iOS 16 中见过的新功能。

**比如锁屏自定义**。

虽然「个性化」这个话题严格来说的确是 Google 先聊起来的，One UI 可以从原生 Android 这边直接沿用、改良的方案也有不少——但或许是 iOS 16 的锁屏自定义功能实在是太受欢迎了，三星还是选择放着大量可以直接沿用、改良的 Android 个性化方案不管，直接在锁屏这里借鉴了 iOS。

![](https://cdn.sspai.com/2022/10/30/0ec8c1f18b410b668d97568e500c6b70.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/d97ede08d2051c1155a3afd4a4d50a11.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

在 One UI 5 中，我们可以通过长按锁屏空白处的方式来激活锁屏自定义设置，设置界面的布局与 iOS 虽然不是完全一样，但基本上也可以说是大同小异：除了基本的锁屏通知样式、底部快捷方式自定义和壁纸更改，我们还可以在这个界面中更改锁屏时间的样式、字体，以及像 iOS 16 那样为壁纸选择不同风格的滤镜。

![](https://cdn.sspai.com/2022/10/30/b1fb605465eb236eab2f44676e9b3e22.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/f096c101d71d6d4e9e0dfc7e88d0482c.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

虽说提供这些设置也并不是不行，但 One UI 5 的锁屏定制功能不仅上得仓促，细节看上去也欠打磨：比如左上角的「壁纸」和左下角的「相册」几乎就是同一个功能的不同层级；再比如锁屏字体样式仅对时间有效、日期完全不受影响，所以在默认字体之外无论选哪个观感都有些奇怪。

因此这段时间体验下来，这个锁屏自定义功能给我最大的感受其实是只得 iOS 其形却不得其神——三星在相册 AI 功能上整那么多花活，为什么没有做 iOS 那样的锁屏时间抠图和视差效果呢？可能是时间不太够吧。

**来到主屏幕这边，One UI 5 进一步优化了小组件堆叠功能**。相同尺寸的小组件现在不仅可以通过拖放自动堆叠，使用和退出时也能像独立小组件那样拥有顺滑的动画效果了。

![boxcnMfQ9ZnasE24DUgGcHWVq02](https://cdn.sspai.com/editor/u_/cdf3ai5b34tf2bu61ghg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

相同尺寸小组件拖拽即可成组

![boxcn6DTbYR5EdeuvsEvAOmAp5d](https://cdn.sspai.com/editor/u_/cdf3aitb34tf3i7qq89g?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

堆叠的小组件也能正常播放进入和退出动画了

**同时，One UI 5 也提供了一个类似 iOS「Siri 建议」那样的智能建议小组件**。这个小组件同样也给人一种仓促感：它将很多系统都有的应用建议（一般是根据用户使用时间和频率生成）以小组件的方式独立出来，支持 4×1 和 4×2（默认）两种尺寸；但不仅无法根据用户的主题图标设置进行图标样式调整，图标数量（一行 5 个）也与用户当前的主屏图标布局完全不同，可以说完全没有任何设计一致性方面的考量。

![](https://cdn.sspai.com/2022/10/30/36c5ae8f0d3a3411ad4ddfe73e69c07d.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/7e6309e387ece4a68f50dbc3ed24eb9c.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

**至于 One UI 5 的重头戏「模式和日常程序」，则是对此前 Bixby 日常程序和 Routines+ 的功能整合，外加一些 iOS 那边的系统模式自动化思路**。

新的「模式和日常程序」有两个入口，一个位于系统设置的一级菜单，另一个则与大部分三星内置应用一样，可以在主界面右上角的三点菜单中开启主屏幕图标显示。但有意思的是，因为被整合到了系统设置中，「模式和日常程序」也是为数不多的几个可以同时存在于两个分屏区域的应用之一。

![boxcneYKKoL2vzT6xhXmjJxMRch](https://cdn.sspai.com/editor/u_/cdf3ak5b34tf3aqo2o80?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

「模式和日常程序」主界面，两个入口进入的界面是可以同时分屏的

其中，模式提供一些基于当前操作和使用场景的自动化设置规则，如睡眠模式下开启灰阶显示、黑暗模式、护眼模式并启用勿扰，驾驶模式可以将特定蓝牙设备连接作为触发手段，屏蔽导航过程中可能会对驾驶造成干扰的无关通知，工作模式则可以设置为在特定时间段内只接受微信、飞书等关键应用的消息推送……

![](https://cdn.sspai.com/2022/10/30/951d95b9221b28df1e21f87341696e51.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/25360f4695e70c320e574ec96b5c802d.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

日常程序部分相比 One UI 4 中的 Bixby 日常程序则几乎没有变化。

![](https://cdn.sspai.com/2022/10/30/5442e08b262edf8359a3e4e9be64a244.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)![](https://cdn.sspai.com/2022/10/30/ed48820d52404d3fc091f341752141b8.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1)

**当然了，整个 One UI 5 最让我感到意外的还是通知系统交互的改动**。One UI 5 在原生特性不多、只能借鉴 iOS 的过程中那种迷茫感，也在这个地方体现得淋漓尽致。

One UI 是为数不多的，对 Android 通知系统交互、设计都没有做太多「魔改」的定制系统（这也是我个人选择三星手机的原因之一）。在 One UI 4 中，我们可以长按任意一条通知并点击设置，然后就能跳转到「通知类别」页面；此时 One UI 4 会像原生 Android 那样，将我们长按的那条通知所对应的通知类别高亮、闪烁，以便我们有针对性地对这一类通知进行管理。
