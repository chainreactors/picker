本文探讨了美国情报界在开源情报（OSINT）领域的现状与问题，并提出了改进建议。作者Chris Rasmussen指出，当前许多针对OSINT的改进计划往往过于关注将OSINT作为收集工作的补充，而不是视其为独立的分析学科。文章强调了OSINT专业性建设的重要性，并提出了如何改变IC内对OSINT的思维和操作模式。文章强调了OSINT必须作为一个独立且专业化的分析领域来发展，而不只是作为分类情报的辅助。美国情报界需要重新设计工作流程，重视OSINT产品的官方发布，并通过集中的OSINT专家团队来促进专业化进程。同时，应减少对最新技术的追逐，更多地投资于人力和分析能力，从而提高OSINT的整体质量和效率。最终，通过建立强大的OSINT专业文化和结构，才能真正实现OSINT在情报界中的地位提升。

+ 政府出版物与开源情报的专业化发展
+ - 开源情报（OSINT）专业性发展的挑战: 许多关于OSINT未来发展的提议主要将其视为一种收集挑战，这限制了OSINT作为完整分析学科的专业化发展。
+ - 专业词汇和孤岛建设的重要性: 所有商业和努力，无论公共或私人，盈利或非盈利，任务驱动，都会围绕其任务执行和劳动形成专业化和共享词汇。
+ 孤岛建设帮助专业化团队沟通和协调。
+ - 缺乏旗舰出版物影响OSINT的专业地位: 相比其他情报类型（INT），OSINT缺乏许多专业要素，例如缺乏旗舰产品。
+ 没有旗舰出版物，无法实现专业的专业化。
+ - 改变当前优先级和流程: 要改变当前的以分类为优先的流程，需要重新设计原则，以便OSINT真正成为第一个分析选择的主导力量。
+ - 重视分析和专业产出: 应该集中精力编写并发表正式的OSINT报告，只引用未分类来源的判断，然后通过正式的质量控制过程。
+ - 建立ODNI托管的OSINT产品线: 创建类似于CIA的WIRe或DIA的国防情报文摘的产品线，将在未分类网络上推动更广泛的OSINT专业化。

+ 改善开源情报（OSINT）操作的挑战与策略
+ - 开源情报（OSINT）专业性的缺乏: 许多提案将OSINT主要视为收集挑战，这降低了OSINT作为分析学科的专业地位。
+ - 过度使用任务整合术语: 这阻碍了OSINT的专业化进程，需要更多OSINT领域的专门团队来改进其操作。
+ - 缺乏旗舰产品和专业标准化: 相较于其他情报类型（INT），OSINT缺乏专业标准化的元素，例如缺乏标志性产品。
+ - 重视整合而非专业性: 专注于整合而不是建立更多的OSINT专门团队，阻碍了OSINT成为正式的分析学科。
+ - 改变工作流程和劳动力利用: 美国情报界（IC）拥有大量劳动力资源，应重新想象工作流程，以OSINT生产为基础，推动更多OSINT工作。
+ - 创建官方的OSINT出版物: 建立类似于CIA的WIRe或DIA的国防情报摘要的ODNI托管的OSINT产品线，以提升OSINT的专业地位。
+ - 扩大OSINT生产的范围: 将OSINT分析师与分析人员的工作集中在未分类领域，从而提高OSINT的专业化和规模效应。
+ - 重新评估工具和技术: 转向更重视分析基础，减少对最新数据管理技术的关注，推动OSINT的专业化发展。
