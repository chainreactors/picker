![]()

微软表示，长达5小时的微软365全球停机是由路由器 IP 地址更改导致的，该更改导致了其广域网中所有其他路由器之间的数据包转发问题。

中断是由 WAN 更新引起的 DNS 和 WAN 网络配置问题造成的，受影响基础设施服务的所有地区的用户在访问受影响的 Microsoft 365服务时遇到问题。该问题导致服务受到波及，在 Microsoft Azure 服务状态页面上共享时大约每30分钟达到峰值（此状态页面也受到影响，因为它间歇性地显示“504网关超时”错误）。[[阅读原文]](https://www.bleepingcomputer.com/news/microsoft/massive-microsoft-365-outage-caused-by-wan-router-ip-change/)
