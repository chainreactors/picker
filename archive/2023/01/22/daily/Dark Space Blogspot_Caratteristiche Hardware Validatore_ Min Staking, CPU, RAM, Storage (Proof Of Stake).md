### Caratteristiche Hardware Validatore: Min Staking, CPU, RAM, Storage (Proof Of Stake)

[![](https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEikkzNE9kK4_3BU81pIeAHMaFeb4DDZdc1CPZpOpxAEi_6qV2z_eASUUlLGhRHiGJUkdHDRFn9yxKH2RuCGVUFxqWMDAUnEGqxiTSx5q6Fpg346IIR_EfdSbGJii_QMnIql-opVcdd5zFGBOknFcV4zqClScr0BUmkT-KE6Uoi0_PlzZqor2OX2UIkL/w400-h266/CPU.jpg)](https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEikkzNE9kK4_3BU81pIeAHMaFeb4DDZdc1CPZpOpxAEi_6qV2z_eASUUlLGhRHiGJUkdHDRFn9yxKH2RuCGVUFxqWMDAUnEGqxiTSx5q6Fpg346IIR_EfdSbGJii_QMnIql-opVcdd5zFGBOknFcV4zqClScr0BUmkT-KE6Uoi0_PlzZqor2OX2UIkL/s275/CPU.jpg)

In questo articolo vedremo quali sono i requisiti minimi per far partire un full node (validatore) sulle principali blockchain con algoritmo di consenso [**Proof Of Stake**](https://darkwhite666.blogspot.com/2020/12/il-proof-of-stake-e-il-dpos-ruoli-dei.html). Ricordo che per queste blockchain, lo staking è fondamentale per validare transazioni e per mettere in sicurezza un network. Le cripto sono pre-minate (a differenza dell'algoritmo di consenso di Bitcoin che è il **[Proof Of Work](https://darkwhite666.blogspot.com/2020/01/qual-e-la-differenza-tra-proof-of-work.html)**). Questi pre-requisiti hardware influenzano la decentralizzazione di una blockchain: tanto più è difficile diventare validatore e costoso mantenere un nodo, tanto più una blockchain tende ad essere meno decentralizzata. Poi ci sono le eccezioni, ad esempio Solana ha circa un migliaio di nodi (non pochi) ma è forse la blockchain con i requisiti hardware più alti (anche se per la verità sono pochi i validatori a validare le transazioni). Ci sono anche blockchain che hanno un numero massimo di validatori (ad esempio Cosmos che ne ha 150). Un altro aspetto da tenere in considerazione è avere una connessione veloce. Ovviamente queste caratteristiche possono variare nel tempo quindi sono solo indicative e non definitive. I parametri da tenere in considerazione sono:

-**CPU** (si riferisce al processore quindi all'elaborazione delle operazioni sotto forma di informazioni. Inizialmente per migliorare le prestazioni, venia semplicemente aumentata la frequenza della CPU, tuttavia questo portava un maggiore surriscaldamento e consumo di energia quindi si è scelto di aumentare il numero dei core. Il core è appunto un componente fisico della CPU ed è l'unità più piccola di elaborazione in grado di svolgere i calcoli necessari all'esecuzione di un certo compito. **[Dual](https://darkwhite666.blogspot.com/2013/10/cosa-significa-dual-core.html)**, **[Quad](https://darkwhite666.blogspot.com/2013/10/cosa-significa-quad-core.html)**ed Octo core è il numero di processori in grado di dividersi un certo compito quindi una CPU multi-core raggiunge prestazioni migliori, consuma meno energia e produce meno calore)

-**RAM** (Random Access Memory è una memoria volatile in cui vengono immagazzinate le informazioni per l'esecuzione di un programma. All'aumentare della **[RAM](https://darkwhite666.blogspot.com/2012/11/memtest-testare-la-ram.html)**aumentano le applicazioni che è possibile avviare contemporaneamente senza che il device si blocchi o rallenti)

-**Storage SSD** (con **[SSD](https://darkwhite666.blogspot.com/2018/09/qual-e-la-differenza-tra-hdd-e-ssd.html)**ci si riferisce allo spazio di archiviazione. Successori degli HDD, sono più veloci, leggeri e duratori. Altra differenza è che usano meno l'alimentazione)

-**Balance** (è il minimo da mettere in staking per avviare un nodo validatore ed è variabile in base alla blockchain)

Vediamo le principali:

[**Ethereum**](https://darkwhite666.blogspot.com/2022/07/cosa-cambiera-con-il-merge-di-ethereum.html): CPU Cores (4), RAM (8-16 GB), SSD (600 GB), Balance (32 ETH + fees)

[**Binance Smart Chain**](https://darkwhite666.blogspot.com/2021/02/guida-completa-per-la-binance-smart.html): CPU Cores (8), RAM (16 GB), SSD (1000 GB), Balance (10.000 BNB)

[**Solana**](https://darkwhite666.blogspot.com/2021/08/solana-il-funzionamento-della.html): CPU Cores (12-16), RAM (128-256 GB), SSD (2000 GB), Balance (1.1 SOL/Day)

[**Cosmos**](https://darkwhite666.blogspot.com/2022/01/tutte-le-funzionalita-di-cosmos-ibc.html): CPU Cores (4), RAM (16 GB), SSD (500 GB), Balance (21.500 ATOM)

[**Avalanche**](https://darkwhite666.blogspot.com/2022/05/avalanche-e-le-subnet-miglior-soluzione.html): CPU Cores (4-8), RAM (4-16 GB), SSD (200 GB), Balance (2000 AVAX)

Polygon: CPU Cores (8), RAM (32 GB), SSD (2500 GB), Balance (1 MATIC)

Cardano: CPU Cores (4), RAM (16 GB), SSD (160 GB), Balance (500 ADA)

[**Polkadot**](https://darkwhite666.blogspot.com/2021/11/come-funzionano-le-aste-di-polkadot.html): CPU Cores (4-8), RAM (32-64 GB), SSD (500 GB), Balance (350 DOT)

[**Cronos Chain**](https://darkwhite666.blogspot.com/2021/11/come-funziona-la-cronos-chain-ibc.html): CPU Cores (4), RAM (4 GB), SSD (25 GB), Balance (No Min Stake)

[**VeChain**](https://darkwhite666.blogspot.com/2022/08/le-applicazioni-reali-e-i-casi-duso-di.html): CPU Cores (4), RAM (8 GB), SSD (500 GB), Balance (25.000.000 VET)

[**Tron**](https://darkwhite666.blogspot.com/2022/05/come-funziona-usdd-su-tron-signoraggio.html): CPU Cores (8-16), RAM (32 GB), SSD (1500 GB), Balance (9.999 TRX)

Near: CPU Cores (4-8), RAM (8 GB), SSD (200 GB), Balance (67.000 NEAR)

Aptos: CPU Cores (4), RAM (8 GB), SSD (300 GB), Balance (1.000.000 APT)

Algorand: CPU Cores (2), RAM (4 GB), SSD (100 GB), Balance (0.1 ALGO)

Harmony: CPU Cores (4), RAM (16 GB), SSD (1500 GB), Balance (10.001 ONE)

Fantom: CPU Cores (4), RAM (16 GB), SSD (1500 GB), Balance (500.000 FTM)

[**Mina**](https://darkwhite666.blogspot.com/2022/09/il-funzionamento-di-mina-protocol-la.html): CPU Cores (8), RAM (16 GB), SSD (100 MB), Balance (-)

La blockchain dai requisiti più elevati è sicuramente Solana. Fanno da contraltare Algorand in generale e Mina Protocol per quanto riguarda lo spazio di archiviazione (questa blockchain pesa solo 21 KB!).
