![Linux terminal Android]()

谷歌最近在安卓开源项目（AOSP）中添加了一项名为 “ferrochrome-dev-option ”的功能，在开发者设置中加入了 “Linux终端 ”选项，允许开发者在安卓环境的虚拟机中运行Linux终端应用程序。

鉴于安卓操作系统原生集成了 Linux 内核，理论上在执行 Linux 终端应用程序时受到的限制应该很小。然而，出于系统一致性、安全性和其他商业因素的考虑，谷歌对安卓操作系统施加了一些限制，使其难以直接或轻松地运行Linux终端应用程序。

随着 Android 开源项目的最新调整，开发者不仅可以在 Android 操作系统中激活 Linux 终端选项，还可以通过虚拟机运行 Linux 应用程序。

这一修改似乎不仅适用于安卓设备，也适用于 Chromebook，有可能使更多 Linux 终端应用程序与 Chrome 浏览器或 Chrome OS 兼容。

鉴于 Linux 开发者社区的庞大规模，谷歌的目的可能是吸引更多 Linux 平台的开发者，从而扩大 Android 上应用程序和应用的多样性。这一调整最早可能会在 Android 16 更新中出现，不过最终的执行情况还要看谷歌后续的修改和决定。
