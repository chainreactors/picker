# 超过30%的恶意攻击以影子API为目标

星阑科技

![](https://mmbiz.qpic.cn/mmbiz_gif/Cc8QqLUKOeiaFHTFtiatmEIxZQcXOHfyr6GOBM88IeMm28ybjSAHEJKicuQxPxN5L5NFZ5mza2NOnuokf9ant2fUQ/640?wx_fmt=gif)

Cequence Security今年发布了其2022年上半年报告，题为“API保护报告：影子API和API滥用爆炸”。调查结果中的主要发现是大约50亿（31%）恶意交易针对未知，未托管和不受保护的API，通常称为影子API，使其成为挑战该行业的最大威胁。

“现实情况是，作为消费者，我们享受的日常奢侈品，如拼车和食品配送服务，都是建立在API之上的，”Cequence Security首席执行官兼创始人Ameya Talwalkar说。“我们的研究发现，公司改善客户体验的创新方式也是对其安全性，客户信任以及最终其底线的最大威胁。这些公司必须从API保护开始，重新考虑其安全策略中的优先级。

该报告由CQ Prime威胁研究团队开发，基于对2022年上半年观察到的超过200亿笔API交易的分析，旨在突出当今困扰组织的主要API威胁。

**主要威胁1：影子 API 被 50 亿个恶意请求击中**

在观察到的 167 亿个恶意请求中，大约有 50 亿 （31%） 针对的是未知、未托管和不受保护的 API（通常称为影子 API），这些请求跨越了广泛的用例。从试图抢夺最新扣篮或Air Jordans的高容量运动鞋机器人，到试图对被盗信用卡进行缓慢的卡测试欺诈的隐形攻击者，再到纯粹的暴力凭证填充活动。在大批量内容抓取作为购物机器人和礼品卡攻击前兆的推动下，对影子API的攻击在2022年4月激增，并且全年数量持续上升。

**主要威胁2：API 滥用**

根据CQ Prime威胁研究团队阻止的36亿次攻击，2022年上半年缓解的第二大API安全威胁是API滥用，这意味着攻击者的目标是正确编码和库存的API。这一发现凸显了使用OWASP等行业标准列表作为起点而不是最终目标的必要性。最常被阻止的攻击表明攻击者正在使用的策略。

**这些包括：**

* 30亿个针对运动鞋或奢侈品的购物机器人
* 2.9亿次礼品卡检查攻击
* 试图在流行的约会和购物应用程序上创建大约2.37亿个虚假帐户

**主要威胁3邪恶的三位一体：撞库，影子API和敏感数据暴露**

基于1亿次攻击，结合使用API2（用户身份验证中断）、API3（数据过度泄露）和API9（资产管理不当）意味着两件事：攻击者正在对每个API的工作原理、它们如何相互作用以及预期的结果进行详细分析，开发人员需要在遵循API编码最佳实践时保持警惕。

**账户接管缓解节省 1.93 亿美元**

CQ Prime 威胁研究团队突显了帐户接管 （ATO） 的持续普及，帮助客户缓解了大约 11.7 亿个恶意帐户登录请求 - 全部针对 API。ATO的受欢迎程度可以直接与其多功能性联系在一起，而采用API进行帐户登录已经放大了这种多功能性，并在本报告中得到了展示。更重要的是，ATO对业务的影响是巨大的，每个事件的成本从290美元（瞻博网络研究）和大约9个小时的调查工作到311美元（联邦贸易委员会）不等。缓解措施保护了大约 1170 万个帐户，相当于为所有客户节省了 1.93 亿美元。

“我们的分析和发现是基于野外的真实攻击，”Cequence安全威胁研究主任威廉·格雷泽（William Glazier）说。“我们的研究结果强调了IT和安全领导者对正确编码的API以及有错误的API如何受到攻击的重要性。仅200亿的样本量就意味着各行各业的企业很有可能受到这些类型的威胁的影响。

该报告强调了了解攻击者用于利用风险的策略、技术和程序 （TTP） 以及攻击者对抵抗的反应的重要性。这意味着不仅要确保API不容易受到OWASP API安全前10名的影响，还要考虑可以定义为API10 +的内容，该类别包含完美编码的API可能被滥用的许多不同方式。

一个严峻的事实是，API发展到现在，授权认证体系已经比较完善，但是在授权之后访问的控制相对薄弱。管控的颗粒度因API接口业务需要而不同，在带来访问便利的同时，也可能被恶意利用，带来信息泄漏和被滥用的风险。API设计之初就是为程序调用准备的，天然是工具行为，利用自动化工具通过合法授权下的API滥用,已成为API攻击的难题。从API的提供方角度，为使用和管理的方便，过度的API开放和宽泛的API调用参数返回，既可能被恶意利用，也可能无形中造成信息泄漏和被滥用的风险。因此传统API安全网关提供的身份认证、权限管控、速率限制、请求内容校验等安全机制几乎无用武之地。

例如，身份认证机制可能存在单因素认证、无口令强度要求、密码明文传输等安全隐患，而访问授权机制风险通常表现为用户权限大于其实际所需权限。同时，即使建立了身份认证、访问授权、敏感数据保护等机制，有时仍无法避免攻击者以机器模拟正常用户行为、运用大量代理IP进行大规模攻击等多种方式来避免速率限制。

在如今互联网的开放场景下，API的应用和部署面向个人、企业、组织机构等不同用户群，是外部网络攻击的主要对象之一，因此更需时刻警惕外部安全威胁。针对API的常见网络攻击包括：重放攻击、DDoS 攻击、注入攻击、会话 cookie 篡改、中间人攻击、内容篡改、参数篡改等，这些新型的安全威胁正在变得更加复杂化、多样化、隐蔽化、自动化。

然而，随着API访问环境的愈发开放、API数量的极速攀升，以及API本身的快速变化，早期的防护技术，如基于规则的传统WAF防护技术、API网关的身份认证和鉴权技术，已经无法满足现有对于API接口被滥用、越权访问、僵尸API、信息泄漏等安全问题的防护需求，新一代基于动态技术、Bots识别、行为分析的融合防护体系逐步兴起。
