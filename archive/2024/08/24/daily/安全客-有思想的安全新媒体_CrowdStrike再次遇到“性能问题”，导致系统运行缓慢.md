周四早上，一些IT管理员遭遇了似曾相识的感觉，因为CrowdStrike将性能问题和启动时间滞后归咎于云服务问题，影响了一些欧洲客户。

“CrowdStrike再次来袭，”Reddit上的一位管理员指出，“至少不是在星期五。

虽然可能不需要解释，但客户指的是 7 月那个致命的星期五，当时错误的文件更新无意中导致了可能是历史上最大的 IT 中断。

另一位管理员报告说，他们的组织“整天都在报告性能问题”，包括“运行时的延迟和缓慢”。

然而，对于陷入困境的安全供应商及其客户来说，幸运的是，这次没有出现蓝屏死机，而且这种补救措施似乎也不会破坏任何管理员的周末计划。CrowdStrike表示，它现在已经解决了这个问题，没有什么可担心的。

该安全业务于周四 0850 UTC 发布了原始警报：

CrowdStrike 发现云服务存在性能问题，该问题可能会导致某些客户系统的性能下降和启动时间下降。此问题影响了我们 EU-1 云中的客户。CrowdStrike Falcon 的客户仍然受到保护。

CrowdStrike 已扩展受影响的云服务。传感器性能趋于稳定。CrowdStrike正在继续关注这一情况。

1220 UTC的状态更新表示，性能正在恢复正常，一位发言人告诉*The Register*，它已经解决了这个问题。

该公司发言人表示：“CrowdStrike今天早上发现并解决了一个云性能问题，该问题导致一小部分欧盟云客户的系统延迟。“这与Channel File 291事件无关，所有客户自始至终都受到保护。”

这位发言人拒绝透露有多少客户受到这个问题的影响：“只有名义数量的客户受到影响。

当然，Channel File 291 事件指的是上个月全球数百万台 Windows 计算机启动循环的有缺陷的 Falcon 端点防御软件更新。

CrowdStrike现在面临来自达美航空公司的法律威胁，达美航空公司声称IT崩溃使其损失了超过5亿美元，以及投资者对其软件做出虚假和误导性陈述的集体诉讼。

在本月早些时候的DEF CON上，CrowdStrike总裁迈克尔·森托纳斯（Michael Sentonas）接受了Pwnie奖的史诗般的失败奖，并承认，“我们犯了严重的错误。
