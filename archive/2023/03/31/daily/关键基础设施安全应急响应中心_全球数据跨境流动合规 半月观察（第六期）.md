# **1.美国医疗和技术协会联盟发布健康数据隐私和安全建议**

2023年2月15日，美国医疗保健团体和技术协会联盟发布了《通过数据隐私和患者访问维护消费者对医疗保健的信任》的报告，旨在推进数据隐私保护，确保患者获取健康信息，同时维持消费者对医疗保健的信任。该报告由医疗保健咨询公司Sirona Strategies推动，以2022年12月健康IT领导力圆桌会议为基础，重点讨论了维持消费者信任的整体重要性、现有健康数据隐私监管框架所带来的机遇和挑战，以及联邦政府为解决数据隐私方面的明显差距而采取的行动。据此，该报告建议提高消费者数据共享和存储方式的透明度，限制健康数据收集、共享、销售和使用的基线规则，并明确联邦政府在实施《健康保险可携性和责任法案》（HIPAA）中的职责。

https://iapp.org/news/a/coalition-releases-health-data-privacy-security-recommendations/

# **2. 信息隐私专业协会披露印度咨询公司Arrka“隐私状况”年度报告**

2023年2月16日，信息隐私专业协会（IAPP）披露了2022年印度隐私咨询公司Arrka的年度报告，即《隐私状况：来自印度的移动应用程序和网站》。该报告分析了移动应用程序和网站在个人数据类型，数据最小化、共享个人数据以及对用户的透明度等方面的情况。其中，报告重点关注移动应用程序和网站的个人数据收集问题，同时对以下关键问题发表见解，包括正在收集哪些个人数据以及如何收集，应用程序收集的个人数据是否超过需要，与谁共享个人数据，个人数据共享的透明度，印度应用程序与全球应用程序的隐私保护相比如何，印度数字借贷应用程序对相关指南的遵守程度，印度应用程序在Google Play Store和Apple App Store上发表隐私声明的程度，以及为即将出台的《印度数据保护法案》做出了哪些准备。

https://iapp.org/news/a/arrkas-state-of-privacy-in-india-annual-report-released/

# **3.德国宪法法院判决警方使用监控软件违宪**

2023年2月16日，德国联邦宪法法院判决黑森州和汉堡州警方使用监控软件的行为违背宪法，因为其使用这种措施干涉公民权利的理由达不到宪法所要求的可确认的危险门槛。该判决具有里程碑意义。该案源于两个州的警方使用应用监控软件程序，在犯罪实施之前先行创建犯罪嫌疑人画像，用于预测性警务工作，而且该应有软件由曾受美国CIA、FBI和NSA资助的数据分析公司Palantir提供。提起该案诉讼的德国民权协会（GFF）认为，警方将监控软件用于预测性警务会增加执法人员犯错的风险，并可能产生歧视性问题。此外，GFF对授权使用该软件的法律依据也提出了质疑，并表示黑森州和汉堡州尚未明确警方可用于获取数据的来源，也未明确执法部门可进行数据挖掘的数量和依据。然而，州代表们认为，该软件是预防犯罪的关键，而且也仅是从其他地方收集的数据收集和处理。Palantir公司则表示其仅提供数据分析服务，数据的来源和使用目的都是由客户决定的。最终，德国联邦宪法法院认定该软件侵犯了公民信息自决权，过度干涉了公民权利，违背了宪法规定，判决相关汉堡州法无效。

https://iapp.org/news/a/german-constitutional-court-blocks-police-use-of-surveillance-software/

# **4.美国宾夕法尼亚州总检察长与DNA诊断中心达成40万美元和解**

2023年2月16日，美国宾夕法尼亚州总检察长（AG）宣布，DNA诊断中心未能正确采取合理的数据安全措施来保护消费者的敏感个人信息，违反了《不公平贸易惯例和消费者保护法》。AG指出，自2021年5月以来，DNA诊断中心多次收到网络可疑活动的通知，但直至同年8月，数据安全提供商通知其网络上有危险恶意软件活动迹象时，DNA诊断中心才启动事件响应计划。此外，AG指出第三方法医报告发现了许多安全漏洞，黑客能够利用这些漏洞备份DNA诊断中心的28个数据库，并从其网络中删除数据。这些被盗数据库包含2004年至2012年接受基因检测的宾夕法尼亚州居民的社会保险号码，受影响的人数达12663人，但DNA诊断中心对此并不知情。因此，AG认为DNA诊断中心的行为违反上述法律，但为了更便捷地解决问题，其与DNA诊断中心达成和解，由该中心向AG支付40万美元，并实施旨在保护消费者个人信息的合理安全政策。

https://www.dataguidance.com/news/pennsylvania-ag-announces-400000-settlement-dna

# **5.英国信息专员办公室对非法营销电话罚款20万英镑**

2023年2月16日，英国信息专员办公室（ICO）宣布对电器售后服务公司It’s OK Limited处以20万英镑的罚款，原因是该公司大量拨打非法营销电话，违反了《2003年隐私和电子通信（EC指令）条例》（PECR）第21条。ICO指出，在2019年至2020年的11个月时间内，该公司向在电话优先服务（TPS）注册的人员拨打了1752149个电话。ICO通过TPS收到了总数达71起对该公司的投诉，其中一些投诉是来自老年弱势群体，而另一些投诉人表示该公司试图误导消费者进行不必要的消费。ICO表示，根据PECR的规定，向TPS注册人员进行电话营销是非法的，除非注册人员通知公司其不反对接收此类电话。据此，ICO认定，该公司向未通知公司愿意接听此类电话的注册人员拨打电话的行为，违反了PECR第21条的规定。鉴于上述情况，ICO对该公司处以20万英镑的罚款，但如果该公司在2023年3月16日之前完成支付，相应罚款可能会减少4至16万英镑。

https://www.dataguidance.com/news/uk-ico-fines-its-ok-limited-for-unlawful-marketing

# **6.欧洲数据保护委员会发布关于反对权和删除权的案例摘要主题文件**

2023年2月20日，欧洲数据保护委员会（EDPB）发布了一份题为“反对权和删除权一站式案例摘要”的主题文件。该文件分析了与GDPR第17条（删除权）和第21条（反对权）相关的决定，并分析了从EDPB公共登记簿中筛选出来的2022年8月至11月期间一站式服务（OSS）的最终决定。此外，该文件还展示了各监管机构为共同执行GDPR所作的努力，让公众有机会审阅不同监管机构做出的、涉及数据主体特定权利的最终决定。

# **7.加拿大新斯科舍省隐私专员办公室发布限制员工不当访问数据的指南**

2023年2月，加拿大新斯科舍省信息和隐私专员办公室（OIPC）发布了关于限制员工不当访问数据的指南。在公众将大量的个人信息和个人健康信息委托给新斯科舍省公共机构、市政当局和健康保管人（组织）的情况下，如果没有适当的预防措施，可能会导致员工在未经授权且没有合法处理目的的情况下访问这些信息。因此，OIPC提出以下防止员工不正当访问和窥探数据的建议，包括培养隐私文化；定期进行培训，并落实有关数据访问的政策；确保员工知道非法访问的后果；确保员工只能访问执行工作所需的信息；实施允许个人阻止特定员工访问其信息的措施；建立适当的访问日志；主动监测和/或审计访问日志；识别和检测不适当的访问；调查所有员工访问报告；当某些措施失败时作出适当调整。

https://iapp.org/news/a/nova-scotia-ipc-offers-guidance-for-public-sector-data-snooping/

# **8.美国纽约州向州议会提交《数据主体权利法案》**

2023年2月，美国纽约州向州议会提交了《数据主体权利法案》。该法案修订了《一般商业法》和《州金融法》，允许消费者对企业向第三方出售或披露的个人信息的类别提出要求。该法案特别强调，消费者有权要求收集消费者个人信息的企业向其披露，包括收集的个人数据的类别；出售的个人数据的类别以及接收数据的第三方身份；出于商业目的披露的个人数据类别，以及以此目的接收数据的第三方身份。此外，该法案还规定，消费者有权随时要求企业终止出售其个人数据，以及在消费者决定终止出售其个人数据后的12个月内，企业不得要求消费者再次授权出售个人数据。最后，该法案规定，受到侵犯的消费者可以提起诉讼，要求法定赔偿。

https://www.dataguidance.com/news/new-york-bill-data-subject-rights-introduced-state-0

# **9.意大利数据保护局对Edison Energia公司非法营销罚款490万欧元**

2023年2月21日，意大利数据保护局（Garante）发布对Edison Energia公司处罚490万欧元的决定，原因是该公司以营销和数据分析为目的非法处理个人数据。Garante指出，Edison Energia公司使用其他公司编制的联系人列表，这些公司不仅未获得用户的书面同意，而且没有明确拒绝接受联系的用户清单，因此无法确定Edison Energia公司获取及使用联系人数据的合法性。此外，投诉人还表示该公司在未经同意的情况下拨打电话，且未对拒绝接听电话的请求做出回应，公司网站与应用程序缺乏详细的同意条款选项，隐私政策也漏洞百出。因此，Garante认定Edison Energia的行为违反了GDPR的透明度原则和责任原则。

# **10.肯尼亚数据保护专员就个人数据跨境传输要求发表声明**

2023年2月21日，肯尼亚数据保护专员Immaculate Kassait MBS发布个人数据跨境传输安全保障要求的声明。专员表示，在将个人数据转移到肯尼亚境外之前，控制者或处理者必须采取适当措施确保境外接收方的数据保护安全水平得到保证，且应对所采取的适当保障措施进行记录，并在专员要求时提供给专员。数据传输所记录的文件应包括传输的日期和时间、接收方名称、传输的理由以及对所传输个人数据的说明等信息。最后，专员指出此举旨在通过加强消费者信任保护个人数据和促进电子商务的发展。

# **11.英国信息专员办公室强调会计师在中小企业数据保护中的作用**

2023年2月22日，英国信息专员办公室（ICO）发布了会计师向中小企业（SME）客户应提出的关键问题建议，确保这些企业正确遵守数据保护法规。ICO表示，该建议是源自于ICO与多达60家英国中小企业合作完成的试点项目。该试点项目企业自我评估和发展新计划“中小企业数据要素”，旨在指导企业更好地管理数据的合规性。该项目研究表明，三分之一以上的中小企业信任其会计师提供的建议，五分之一的中小企业积极利用会计师了解最新的数据保护规则。据此，ICO列出了会计师向中小企业提出的七个关键问题，包括企业对数据保护合规性和ICO了解多少？企业每天会收集哪些类型的个人信息？为什么企业要持有个人信息？企业有什么安全措施？企业是否有隐私声明？企业是否了解数据主体的访问请求？企业是否知道个人数据泄露后应该采取哪些措施？ICO表示，希望通过发布该建议，呼吁英国会计师认识到他们在帮助中小企业客户正确遵守数据保护法规的关键作用。

# **12.法国数据保护局讨论电子健康卡对隐私权的影响**

2023年2月22日，法国数据保护局（CNIL）发布了一份新闻稿，对未来可选的电子健康保险卡（E-Carte Vitale）的隐私问题进行说明。CNIL指出，计划在2025年底前向所有健康保险提供商推出E-Carte Vitale健康卡的电子版本，该版本包含实体健康卡中要求的所有管理信息，涵盖与治疗报销、住院保险和持卡人认证相关的信息，但并不会被用于收集医疗数据。关于持卡人认证，CNIL强调，E-Carte Vitale不是生物识别卡，其使用生物识别信息仅用于激活应用程序，不会用于医疗或住院。CNIL注意到，政府为了保护数据主体的隐私权和数据安全已大幅修改了电子健康卡的相关法令草案。

# **15. 以色列隐私保护局澄清《数据安全条例》“第三方”和《隐私保护法》“持有人”的区别**

2023年2月23日，以色列隐私保护局（PPA）通过LinkedIn阐明了《隐私保护（数据安全）条例》（《数据安全条例》）第15条中的“第三方”和《隐私保护法》（PPL）中“持有人”之间的区别。PPA指出，《数据安全条例》第15条中的“第三方”一词比PPL中的“持有人”一词更为宽泛，指的是通过提供服务可能实际访问数据库或其系统的任何一方，即使该访问并非提供服务所必需。PPA称，上述对数据库的访问不仅涉及对存储在数据库中的信息的访问，还涉及对数据库系统的访问；并且就《数据安全条例》第15条而言，若要将访问方视为“第三方”，则无需永久授予访问权限。在大多数情况下，《数据安全条例》第15条规定的“第三方”也将是PPL中定义的“持有人”，并且这两个身份之间会有重叠，例如在第三方获得使用、处理或存储信息的许可的情况下。然而，在某些情况下，第三方不会被视为持有人，例如与第三方签订合同的目的是为了获得与数据库硬件系统相关的服务，或为了纠正数据库软件故障。
