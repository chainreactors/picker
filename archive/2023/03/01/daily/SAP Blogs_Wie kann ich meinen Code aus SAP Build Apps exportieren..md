Die Idee von SAP Build Apps ist ja, dass Fachbereichsanwender (Citizen Developer) enger mit der IT (Professionel Developer) zusammenarbeiten können und sollen, um die aktuellen Anforderungen zügig umsetzten zu können. Je nachdem, was man umsetzten möchte, kann es sein, dass Sie gerne auf den vollständigen Code, den man mit SAP Build Apps erzeugt, Zugriff haben möchten.

Ein Beispiel kann sein, der Fachbereich möchte eine Applikation nutzen und entwirft mit SAP Build Apps das Frontend und fügt einige Logiken hinzu. Die professionellen Developer möchten das Backend und die letzten Schritte aber nicht in einer No-Code Lösung, sondern einer Pro-Code Lösung umsetzten. In einem solchen Fall ist es wichtig, dass man an den Code kommt, den man in SAP Build Apps generiert hat.

Hier findet ihr eine kurze Anleitung wie man im Standard in SAP Build Apps sich seinen Code ausgeben lassen kann (JavaScript und HTML5).
