## [Kubernetes Cryptojacking](/?page=Blog&month=2024-05&post=Kubernetes-Cryptojacking)

In this video I take a look at a cryptojacking attack against a Kubernetes honeypot.
The attackers were surprisingly quick to discover this unsecured Kubernetes deployment and use it to mine Monero for them.

[![

](https://media.netresec.com/images/kubernetes-cryptojacking_1280x720.png)](https://media.netresec.com/videos/kubernetes-cryptojacking_1280x720.mp4)

The analyzed capture files can be downloaded from
<https://share.netresec.com/s/S5ZG2cDKB9AbqwS?path=%2Fk3s-443>

This PCAP dataset was created by Noah Spahn, Nils Hanke, Thorsten Holz, Chris Kruegel, and Giovanni Vigna as part of their research for their [Container Orchestration Honeypot: Observing Attacks in the Wild](https://seclab.cs.ucsb.edu/files/publications/spahn_cohp_23.pdf) paper.

The capture files named "proxy-", such as the analyzed proxy-220404-162837.pcap, were generated by [PolarProxy](https://www.netresec.com/?page=PolarProxy) and contain the decrypted Kubernetes API traffic to the master node.
This traffic was actually TLS encrypted, but since PolarProxy was used as a TLS interception proxy we can see the Kubernetes API traffic in decrypted form.
