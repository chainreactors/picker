## 全球动态

### 1. 135个上榜！2024年网络安全技术应用典型案例项目名单公布

近日，工业和信息化部办公厅、国家互联网信息办公室秘书局、人力资源社会保障部办公厅等13个部门联合公布2024年网络安全技术应用典型案例项目名单。 【[阅读原文](https://mp.weixin.qq.com/s/f82aZsuuPSRIjXR14snPvw)】

### 2. 建立公共数据资源授权运营价格形成机制

为促进公共数据资源合规高效流通使用，近日，国家发展改革委、国家数据局印发《关于建立公共数据资源授权运营价格形成机制的通知》。【[阅读原文](https://finance.huanqiu.com/article/4LMIrC2yu8x)】

### 3. 谷歌AI原则生变：删除“不用于武器”的承诺

谷歌本周从其网站上删除了不开发用于武器或监控的人工智能的承诺。这一变化最初由彭博社发现。该公司似乎更新了其公开的人工智能原则页面，删除了上周还存在的名为“我们不会追求的应用”的部分。【[阅读原文](https://www.ithome.com/0/828/553.htm)】

### 4. DeepSeek又被攻击，本地化部署是否安全？

既然网络有被攻击的风险，很多人就希望本地化部署DeepSeek，那么，本地化部署DeepSeek简单吗?部署完成后就可以避免安全问题发生了吗? 【[阅读原文](https://mp.weixin.qq.com/s/eFxjHAMhD7kmu6ySBiJPDA)】

### 5. 研究人员发现新方法防御AI模型的通用越狱攻击

研究人员开发了一种新方法，用于保护人工智能模型免受通用越狱攻击。这种创新方法被称为“宪法分类器”，已在数千小时的人类红队测试和合成评估中表现出了显著的抗攻击能力。 【[阅读原文](https://cybersecuritynews.com/researchers-uncovers-new-methods-to-defend-ai-models/#google_vignette%EF%BB%BF)】

### 6. 加州拟出新规：AI公司必须定期提醒儿童聊天机器人不是人类

美国加利福尼亚州提出了一项新法案（SB 243），该法案要求人工智能（AI）公司定期提醒儿童，聊天机器人是 AI 而非人类。这项法案由加利福尼亚州参议员史蒂夫・帕迪拉提出，旨在保护儿童免受 AI“容易使人上瘾、造成孤立和产生影响”等方面的危害。 【[阅读原文](https://www.ithome.com/0/828/653.htm)】

## 安全事件

### 1. Windows Server 2025域控制器被曝BUG，导致微软Win11客户端密码更新机制失效

科技媒体 borncity 2月5日发布博文，报道称Windows Server 2025域控制器出现信任关系问题，导致Windows 11客户端密码更新失败。 【[阅读原文](https://www.ithome.com/0/828/667.htm)】

### 2. CISA将四个被积极利用的漏洞添加到KEV目录中

美国网络安全和基础设施安全局 (CISA) 在其已知被利用漏洞 ( KEV ) 目录中添加了四个安全漏洞，并指出有证据表明这些漏洞正在被积极利用。 【外刊-[阅读原文](https://thehackernews.com/2025/02/cisa-adds-four-actively-exploited.html)】

### 3. Chrome的“Use-After-Free ”漏洞让攻击者能够执行远程代码

Google Chrome 团队已正式发布 Chrome 133 ，最新更新解决了12个安全漏洞，其中包括两个高严重程度的释放后使用漏洞，这些漏洞可能允许攻击者利用内存管理问题并远程执行恶意代码。【外刊-[阅读原文](https://cybersecuritynews.com/chrome-use-after-free-vulnerabilities/)】

### 4. 新的Tiny FUD正针对macOS用户，绕过杀毒软件和安全工具

一种名为Tiny FUD的新型、高度复杂的恶意软件已被发现，它针对macOS用户，利用先进的规避技术绕过传统的杀毒软件和安全工具。该恶意软件通过进程名称伪装、DYLD注入以及基于命令与控制（C2）的命令执行来保持隐蔽。【外刊-[阅读原文](https://cybersecuritynews.com/new-tiny-fud-attacking-macos-users/)】

### 5. Coyote Banking Trojan瞄准巴西用户，窃取70多个金融应用程序和网站的数据

研究人员检测到一项活动，该活动使用LNK文件执行PowerShell命令来部署Coyote Banking Trojan。威胁行为者通过窃取财务数据来瞄准巴西用户，该恶意软件可以从70多个金融应用程序和众多网站中收集敏感信息。【外刊-[阅读原文](https://securityaffairs.com/173818/malware/coyote-banking-trojan-targets-brazilian-users.html)】

### 6. 严重的Windows OLE零点击漏洞使攻击者能够执行任意代码

微软的Windows对象链接和嵌入 (OLE) 技术中披露了一个严重的安全漏洞，编号为CVE-2025-21298。

该零点击漏洞的CVSS评分为9.8，允许攻击者通过利用Microsoft Outlook和其他应用程序远程执行任意代码。【外刊-[阅读原文](https://cybersecuritynews.com/critical-windows-ole-zero-click-vulnerability/)】
