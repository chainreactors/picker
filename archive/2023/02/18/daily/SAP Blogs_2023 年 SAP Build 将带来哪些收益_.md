此文章为翻译文，翻译原文为 [What to expect in 2023 from SAP Build](https://blogs.sap.com/2023/01/17/what-to-expect-in-2023-from-sap-build./?source=social-Global-TWITTER-AudienceEngagement-Business_Technology_Platform-SAPBuild-spr-8667029373-account_name&campaigncode=CRM-YA22-BSC-1440506&sprinklrid=8667029373)，原文作者为sebastian.schroetel。

随着 2023 年的开始，我想重点介绍一下我们在 SAP Business Technology Platform 的低代码/无代码产品组合中的一些关键时刻，并重点介绍我们今年取得的一些惊人成就。

但是，对于未来几个季度，我更需要分享一些目标和观点，我很高兴向您介绍一下我们正在为 SAP Build 开展哪些工作。

## **2022 年，我们的低代码/无代码之旅**

2 月，[我们推出了 SAP Build Process Automation](https://blogs.sap.com/2022/02/14/accelerate-automation-with-a-new-no-code-experience-with-sap-process-automation-now-generally-available/)，将 RPA、工作流和人工智能功能整合一体，提供直观的无代码体验，**帮助客户实现业务流程自动化并提高效率**。我们的战略举措得到了市场的认可：在夏季，SAP 在 2022 年 Gartner® 的 Robotic Process Automation Magic Quadrant™ 报告中[被评为远见者](https://news.sap.com/2022/08/sap-visionary-gartner-robotic-process-automation/)，而我们得到了[早期采用者的积极反馈](https://www.linkedin.com/pulse/early-adopter-feedback-confirms-value-citizen-process-schr%C3%B6tel/)，导致 SAP 被客户大量采用。

![](/legacyfs/online/storage/blog_attachments/2023/02/Supalai-SBPA-ref.jpg)

6 月，我们在 SAP AppGyver 中推出了[可视化云功能](https://blogs.sap.com/2022/06/06/introducing-the-full-stack-no-code-experience-with-visual-cloud-functions-for-sap-appgyver/)，现已演化为 SAP Build 应用，引入一种为应用程序可视化创建云部署后端的方法。借助 SAP Build 应用和此类云端编程功能，我们为无代码提供[全栈开发体验](https://blogs.sap.com/2022/12/28/one-tool-no-code-all-magic-mobile-app-and-backend-by-sap-build./)，**每个人都能成为全栈云开发人员**。

最后，在 SAP TechEd 大会上，[我们推出了 SAP Build](https://news.sap.com/2022/11/sap-build-capabilities-under-the-hood-sap-teched/)，这是我们**统一的低代码产品**。它旨在为每种技能水平的用户（包括很少或没有正式编码经验的用户）提供支持，它汇集了三个核心组件

* [SAP Apps](https://community.sap.com/topics/appgyver)，支持以简单的拖放功能创建企业级应用

* [SAP Build Process Automation](https://community.sap.com/topics/process-automation) 可直观地自动化流程和任务

* [SAP Build Work Zone](https://community.sap.com/topics/work-zone)，使用可视编辑器和预构建模板设计业务站点

加速创新的真正亮点在于，感谢我们重要的客户和合作伙伴，感谢他们的参与和支持：您参与了 beta 计划、客户反馈研讨会，并分享了非常有价值和启发性的反馈，帮助我们改进了产品。

## 2023 年的预期是什么？

我们今年的目标很明确：**让 SAP 构建面向 SAP 客户的最佳低代码平台**。

为此，我们为未来几个季度确定了关键优先事项，以培育我们的[路线图](https://roadmaps.sap.com/board?range=CURRENT-LAST&PRODUCT=73554900100800003832&PRODUCT=73554900100800004334&PRODUCT=73555000100800002781#Q1%202023)：

1. **更好地集成我们产品的三大支柱**：例如，这意味着我们希望更轻松地将流程连接到应用程序，并构建您可以嵌入到业务站点中的新应用程序。我们还希望更轻松地在不同解决方案之间共享部件。

2. **借助丰富的业务环境，与 SAP 核心应用深度集成**：事实上，我们将继续以 SAP Build 开发为核心的主题，即与 SAP S/4HANA、SAP SuccessFactors、Ariba 集成……我们将继续将低代码功能嵌入到 SAP 解决方案中，并提供更多预构建的内容。除 ABAP 和 Steampunk 等其他功能外，SAP Build 是一种非常有效的方式，不仅可以扩展 S/4HANA，还可以扩展 SAP SuccessFactors、SAP Ariba、SAP CX 等。

3. **改进了 IT 治理**：尽管我们通过 SAP Business Technology Platform 已经拥有大量功能，但我们希望为 IT 管理员提供更多功能来监控和运行低代码应用和流程。

4. **专业开发人员的更好体验**：我们还将专注于进一步调整专业开发人员的开发体验。我们将持续投资，改进我们面向 IT 专业人员的低代码/无代码产品，并为专业开发人员提供明确指导，说明何时使用什么实现最佳成果。

5. **与 SAP BTP 上提供的高级云技术的深度集成。**我们在这方面的努力包括：

* [SAP Cloud Application Programming Models](https://roadmaps.sap.com/board?q=%20SAP%20Cloud%20Application%20Programming%20&range=CURRENT-LAST#Q2%202023) 将作为服务使用。它将使融合团队的 IT 专家能够构建全面的数据逻辑，并在 SAP Build Process Automation 中加以利用。

* [SAP Graph](https://roadmaps.sap.com/board?q=%E2%80%A2%09SAP%20Graph%20&range=CURRENT-LAST#Q1%202023)将支持 SAP Build 用户直接访问所有 SAP 应用程序中语义连接的数据实体。

* [SAP AI Business Services](https://community.sap.com/topics/artificial-intelligence) 将与 SAP Build Process Automation 越来越集成，允许在设计时和运行时广泛使用 AI。这包括自动识别应用程序更改，以在设计时建议更正。对于运行时间，它包含故障根本原因分析、基于最佳推荐操作的自动化自我修复更正以及请求人文循环或自动调整作业分配。

* [SAP Signavio](https://www.signavio.com/) 将与 SAP Build Process Automation 进一步集成，加速从洞察到行动的业务转型：SAP Signavio Process Intelligence 将通过 API 触发自动操作，以通过工作流和机器人执行流程自动化，SAP Build Process Automation 将补充 SAP Signavio Process Governance，管理 SAP Signavio Process Manager 中流程模型变更的审批。

如你所见，我们正在讨论激动人心的主题，在接下来的几个季度，您将看到 SAP Build 中的更多创新。敬请关注。

祝大家在 2023 年有一个不错的开端，也是一个低代码令人惊叹的一年！
