# New Release: Tor Browser 14.5.2

by [ma1](/author/ma1)
| May 18, 2025

![](/new-release-tor-browser-1452/lead.png)

Tor Browser 14.5.2 is now available from the [Tor Browser download page](https://www.torproject.org/download/) and also from our [distribution directory](https://www.torproject.org/dist/torbrowser/14.5.2/).

This version includes important [security updates](https://www.mozilla.org/en-US/security/advisories/) to Firefox.

## Send us your feedback

If you find a bug or have a suggestion for how we could improve this release, [please let us know](https://support.torproject.org/misc/bug-or-feedback/).

## Full changelog

The full changelog since [Tor Browser 14.5.1](https://gitlab.torproject.org/tpo/applications/tor-browser-build/-/raw/maint-14.5/projects/browser/Bundle-Data/Docs-TBB/ChangeLog.txt) is:

* All Platforms
  + [Bug tor-browser#43397](https://gitlab.torproject.org/tpo/applications/tor-browser/-/issues/43397): Click to play should override "Any capability blocked in the top document must be blocked in its subdocuments too"
  + [Bug tor-browser#43802](https://gitlab.torproject.org/tpo/applications/tor-browser/-/issues/43802): Rebase Tor Browser Stable onto Firefox 128.10.1esr
  + [Bug tor-browser-build#41460](https://gitlab.torproject.org/tpo/applications/tor-browser-build/-/issues/41460): Add brizental to the list of people who can sign Tor Browser and Mullvad Browser tags
* Windows + macOS + Linux
  + Updated Firefox to 128.10.1esr
  + [Bug tor-browser#43635](https://gitlab.torproject.org/tpo/applications/tor-browser/-/issues/43635): Console noise due to the x-load capability not being localized
  + [Bug tor-browser#43738](https://gitlab.torproject.org/tpo/applications/tor-browser/-/issues/43738): Backport Bugzilla 1947157: Move linux security warning content into own page
* Android
  + Updated GeckoView to 128.10.1esr
* Build System
  + All Platforms
    - [Bug tor-browser#43777](https://gitlab.torproject.org/tpo/applications/tor-browser/-/issues/43777): Disable ./mach telemetry
    - [Bug tor-browser-build#41449](https://gitlab.torproject.org/tpo/applications/tor-browser-build/-/issues/41449): Add prefix to update-responses xml files
    - [Bug tor-browser-build#41451](https://gitlab.torproject.org/tpo/applications/tor-browser-build/-/issues/41451): When update-responses contains multiple versions, .htaccess only has one no-update.xml redirect
    - [Bug tor-browser-build#41452](https://gitlab.torproject.org/tpo/applications/tor-browser-build/-/issues/41452): Skip update-responses xml files for versions which don't have incrementals
  + Windows + Linux + Android
    - Updated Go to 1.23.9
